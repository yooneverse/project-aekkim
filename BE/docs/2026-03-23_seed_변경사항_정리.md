## 3. 서비스 카탈로그 변경

### 3.1 변경 내용

- `COUPANG_PLAY.name`
  - `쿠팡` -> `쿠팡플레이`
- `COUPANG_PLAY.customerServicePhone`
  - `1600–9800` -> `1600-9800`

### 3.2 변경 이유

- 티빙 해지 가이드가 온보딩 또는 부정확한 페이지로 연결되는 문제를 수정했다.

### 3.3 재반영 방법

## 4. 번들 프로모션 변경

### 4.1 삭제한 항목

- `더블 (디즈니+)`
- `3 PACK`

### 4.2 유지한 항목

- `디즈니+ 티빙 번들`
- `디즈니+ 티빙 웨이브 번들`
- `더블 광고형 스탠다드`

- `더블 프리미엄`

### 4.2-1 번들 필드 보강

- `bundle.json`의 각 번들 항목에 `originalPrice`를 추가했다.
- `monthlyPrice`는 현재 판매가, `originalPrice`는 할인 전 기준가로 사용한다.
- 이를 통해 번들에서도 `얼마에서 얼마로 할인되는지`, `월 절감액이 얼마인지`를 계산할 수 있다.

### 4.3 변경 이유

- 서비스 조합과 가격이 동일한 별칭성 번들 항목을 제거했다.
- 실제 요금제 구분이 있는 더블 상품군은 유지했다.

### 4.4 재반영 방법

- 관리자 웹 `프로모션` 탭에서 [bundle.json](/C:/Users/SSAFY/workspace/S14P21E106/seed/bundle.json)을 다시 import한다.

## 5. 카드 혜택 프로모션 변경

### 5.1 summary 정제

- 카드 혜택 프로모션의 `summary` 원본 문자열을 정제했다.
- 주요 정제 규칙:
  - `?` 제거
  - `전월실적20만원 이상` -> `전월실적 20만원 이상`
  - `VISAMastercard` -> `VISA / Mastercard`
  - 줄바꿈 기준으로 `연회비`, `국내전용`, `해외겸용`, `전월실적`, `브랜드`가 읽히도록 정리

### 5.2 정제 예시

기존:

```text
연회비
국내전용 없음  해외겸용 없음?전월실적20만원 이상
브랜드
VISAMastercard
```

변경 후:

```text
연회비
국내전용 없음
해외겸용 없음
전월실적 20만원 이상
브랜드
VISA / Mastercard
```

### 5.3 import 정책 변경

- `services: []` 인 프로모션은 seed에 유지하되, import 시 DB에 적재하지 않도록 처리했다.
- 즉 원본 JSON은 보존하고, 서비스 연결 없는 카드 혜택만 DB 적재 대상에서 제외한다.

### 5.4 재반영 방법

- 관리자 웹 `프로모션` 탭에서 [card_benefits.json](/C:/Users/SSAFY/workspace/S14P21E106/seed/card_benefits.json)을 다시 import한다.

## 6. 날짜/이미지 필드 규칙

- 프로모션 seed의 날짜 필드는 offset 없는 `LocalDateTime` 문자열을 사용한다.
  - 예: `2026-03-22T22:21:55.268463`
- 카드 이미지 필드명은 `imgUrl`이 아니라 `imageUrl`을 사용한다.

## 7. 백엔드 반영 사항

- [AdminSeedImportService.java](/C:/Users/SSAFY/workspace/S14P21E106/BE/src/main/java/com/ssafy/e106/domain/admin/service/AdminSeedImportService.java)
  - seed 값을 그대로 import하는 구조로 유지
  - 단, `services: []` 인 프로모션은 import 시 스킵

## 8. 운영 메모

- seed 파일 수정 후에는 DB 값이 자동 반영되지 않는다.
- 관리자 웹에서 대상 JSON 파일을 다시 import해야 최신 값이 반영된다.
- 서비스 seed 수정 시 `서비스` 탭에서 재import
- 프로모션 seed 수정 시 `프로모션` 탭에서 재import
