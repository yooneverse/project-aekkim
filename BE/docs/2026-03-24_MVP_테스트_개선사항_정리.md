## 1. 알림 권한 및 수신 토글 의미 불일치

### 고칠 것
- 마이페이지의 알림 토글이 시스템 알림 권한과 같은 의미처럼 보여 혼선이 있었다.
- 시스템 알림은 허용되어 있어도 앱 내 토글은 꺼져 보일 수 있어 사용자가 상태를 이해하기 어려웠다.

### 해결 방안
- 마이페이지에 `시스템 알림 권한` 상태를 별도로 표시한다.
- 체크인/혜택 토글은 앱 내 알림 수신 설정이라는 의미가 드러나도록 안내 문구를 추가한다.
- 시스템 알림 권한이 꺼져 있을 경우 설정 화면으로 바로 이동할 수 있게 연결한다.

## 2. 구독 상세 추천 금액 문구 문제

### 고칠 것
- 구독 상세 추천 영역에서 `매월 N원 절약하는 방법`처럼 과하게 읽히는 문구가 표시되었다.
- 연간 프로모션 총 할인액이 월 절감처럼 보여 오해 소지가 컸다.

### 해결 방안
- 프로모션에 `billingCycle`을 추가해 `MONTHLY`, `YEARLY`를 구분한다.
- 구독 상세 추천 응답에도 `billingCycle`을 포함한다.
- 추천 문구는 `월 최대/연 최대 ... 할인 가능` 형태로 완화한다.

## 3. 구독 상세 `추천 보기` 이동 버그

### 고칠 것
- 구독 상세에서 `추천 보기`를 누르면 해당 추천의 상세 페이지가 아니라 추천 목록 탭으로 이동했다.

### 해결 방안
- `추천 보기` 버튼이 현재 추천 프로모션의 상세 화면으로 이동하도록 수정한다.
- `promotionId`를 상세 route로 직접 전달한다.

## 4. 티빙 해지 가이드 리다이렉트 이슈

### 고칠 것
- 티빙 해지 가이드 URL은 seed/DB 기준 정상인데, 모바일 환경에서 온보딩처럼 보이는 페이지로 리다이렉트되는 현상이 있다.
- FE가 URL을 잘못 여는 문제보다는 외부 웹 동작 가능성이 높다.

### 해결 방안
- 현재는 보류한다.
- 시연 직전 다시 확인하고, 필요하면 더 안정적인 티빙 고객센터 URL을 찾는다.
- 외부 웹 리다이렉트 이슈로 보이는 만큼 FE 수정은 우선순위를 낮춘다.

## 5. 연간 요금제 가격 및 결제일 표시 문제

### 고칠 것
- 연간 요금제가 `4,158원`, `7,916원`처럼 월간 요금제처럼 보였다.
- 구독 추가/수정 화면에서도 월간/연간 구분이 약해 사용자가 혼동할 수 있었다.
- 연간 요금제의 결제일은 `매월 N일`처럼 수정하는 구조와 맞지 않았다.

### 해결 방안
- 구독 상세와 구독 추가/수정 화면에서 `월간/연간` 라벨을 함께 표시한다.
- 월간 요금제는 `월 N원`, 연간 요금제는 `월 환산 N원` 형태로 의미가 드러나게 노출한다.
- 연간 요금제를 선택한 경우 결제일은 읽기 전용 안내로만 보여준다.

## 6. 관리자 결제내역 탭 API 불일치

### 고칠 것
- 관리자 페이지 결제내역 import/조회 UI가 여전히 `userId` 기반으로 되어 있었다.
- 백엔드는 이미 공용 결제내역 구조로 바뀌어 실제 계약과 불일치했다.

### 해결 방안
- 관리자 결제내역 import/조회 폼에서 `userId` 입력을 제거한다.
- 공용 결제내역 API 계약에 맞게 호출을 수정한다.

## 7. 구독 수정 네비게이션 상태 중복

### 고칠 것
- 구독 수정 진입 상태를 repository와 navigation이 동시에 관리하고 있었다.
- 이 상태가 계속 남아 있으면 이후 다시 네비게이션 충돌이 생길 수 있다.

### 해결 방안
- 구독 수정 대상 전달은 `savedStateHandle` 한 가지 방식으로만 통일한다.
- repository에 남아 있던 임시 수정 상태와 관련 메서드는 제거한다.

## 8. 신규 회원 알림 토글 기본값 문제

### 고칠 것
- 신규 회원 생성 시 체크인/혜택 알림 기본값이 `false`라서 초기 경험에서 토글이 꺼져 보였다.
- 시스템 알림 권한을 허용해도 마이페이지는 OFF처럼 보여 혼선이 생겼다.

### 해결 방안
- 신규 회원 생성 시 체크인 알림, 혜택 알림 기본값을 `true`로 변경한다.
- 시스템 권한과 앱 내 토글 의미를 분리해 화면에 표시한다.

## 9. 카드 혜택 상세 텍스트 중복 노출 문제

### 고칠 것
- 카드 혜택 상세 화면에서 같은 summary가 상단 설명 카드와 하단 `혜택 확인` 카드에 중복 노출되고 있다.
- 사용자는 같은 정보를 두 번 보게 되고, 카드 혜택 화면이 더 복잡하게 느껴진다.

### 해결 방안
- 카드 혜택 summary는 하단 `혜택 확인` 카드에만 남긴다.
- 상단 상세 영역에서는 카드 혜택 summary를 제거해 중복 노출을 없앤다.
- 하단 카드의 renderer만 유지해 혜택 정보 표시 규칙을 한 곳으로 통일한다.
## 10. 카드 혜택 정렬 기준 확정 및 반영

### 고칠 것
- 카드 혜택 추천이 절감액 중심으로만 섞여 보여 실제로 바로 체감되는 혜택이 뒤로 밀릴 수 있음
- `할인`, `적립`, `%`, `원`처럼 사용자가 즉시 이해할 수 있는 혜택 정보가 드러나는 항목을 우선해서 보여줄 필요가 있음

### 해결 방안
- 카드 혜택은 `할인/적립/%/원` 키워드가 제목 또는 summary에 드러나는 항목을 우선 노출
- 그 다음 추천 점수, 절감액, 최신순 기준으로 정렬
- 프론트에서 임의 정렬하지 않고 추천 점수와 카드 혜택 우선순위가 반영된 순서를 유지

## 11. 추천이 왜 추천인지 체감되도록 추천 표현 및 구조 보완

### 고칠 것
- 추천 카드가 제목과 요약만 보여주면 왜 지금 사용자에게 추천되는지 바로 이해하기 어려움
- 카드 혜택, 프로모션, 번들 모두 추천 사유가 한눈에 보이지 않아 추천 체감이 약함

### 해결 방안
- 추천 API에 추천 점수와 추천 이유 문구를 함께 내려주도록 확장
- 추천 카드 상단에 이유 칩을 1~2개 노출해 추천 사유를 바로 인지할 수 있게 구성
- 추천 이유는 절감 가능성, 현재 사용 서비스 일치, 결제일 임박, 최근 사용 이력, 최근 등록 혜택 여부 등을 기준으로 생성
