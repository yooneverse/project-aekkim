# 데모운영용 관리자 웹 구현 계획

## 1. 문서 목적

- 데모 진행에 필요한 기준 데이터 import, 조회, 결제내역 import, 알림 발송, 배치 실행을 운영자가 직접 제어할 수 있도록 관리자 웹 구성을 정리한다.
- 현재 구현은 데모 운영 목적에 맞춘 최소 기능만 남긴 상태이며, 일반 사용자용 관리 기능과는 분리한다.

## 2. 현재 구현 범위

### 2.1 서비스 운영

- 서비스 / 요금제 JSON import
- 서비스 / 요금제 조회

### 2.2 프로모션 운영

- 프로모션 JSON import
- 프로모션 조회

### 2.3 결제내역 운영

- 결제내역 JSON import
- 사용자 기준 결제내역 조회

### 2.4 알림 운영

- 수동 알림 발송

### 2.5 배치 운영

- CHECKIN 대상 선정 배치 수동 실행
- CHECKIN 발송 배치 수동 실행
- PROMO 대상 선정 배치 수동 실행
- PROMO 발송 배치 수동 실행

## 3. 제외한 기능

- 서비스 CRUD
- 요금제 CRUD
- 프로모션 CRUD
- 저사용 상태 수동 수정

## 4. 현재 관리자 API

### 4.1 서비스 API

- `POST /api/v1/dev/admin/imports/service-catalog`
  - multipart 필드명: `file`
  - 서비스 / 요금제 JSON 1건 업로드
- `GET /api/v1/dev/admin/services`
  - 서비스 / 요금제 조회

### 4.2 프로모션 API

- `POST /api/v1/dev/admin/imports/promotions`
  - multipart 필드명: `files`
  - 프로모션 JSON 1건 이상 업로드
- `GET /api/v1/dev/admin/promotions`
  - 프로모션 조회

### 4.3 결제내역 API

- `POST /api/v1/dev/admin/payment-histories?userId={id}`
  - multipart 필드명: `file`
  - 결제내역 JSON 1건 업로드
- `GET /api/v1/dev/admin/payment-histories?userId={id}`
  - 사용자 기준 결제내역 조회

### 4.4 알림 API

- `POST /api/v1/dev/admin/notifications`
  - 최신 FCM 토큰이 있는 사용자 기준 수동 알림 발송

### 4.5 배치 실행 API

- `POST /api/v1/dev/admin/batch-executions/checkin-target-selection`
- `POST /api/v1/dev/admin/batch-executions/checkin-notification-send`
- `POST /api/v1/dev/admin/batch-executions/promo-target-selection`
- `POST /api/v1/dev/admin/batch-executions/promo-notification-send`

## 5. 관리자 페이지 구성

### 5.1 탭 구성

1. `서비스`
2. `프로모션`
3. `결제내역`
4. `수동 알림`
5. `배치 실행`

### 5.2 서비스 탭

- 서비스 / 요금제 JSON 파일 1개 선택 후 import
- 서비스 / 요금제 조회

### 5.3 프로모션 탭

- 프로모션 JSON 파일 1개 이상 선택 후 import
- 프로모션 조회

### 5.4 결제내역 탭

- `userId + 결제내역 JSON` 업로드
- 사용자 기준 결제내역 목록 조회

### 5.5 수동 알림 탭

- `userId`
- `type`
- `referenceId`
- `title`
- `body`

입력 후 즉시 발송

### 5.6 배치 실행 탭

- CHECKIN 대상 선정
- CHECKIN 발송
- PROMO 대상 선정
- PROMO 발송

각 Job을 버튼으로 즉시 실행

## 6. 현재 사용 방법

### 6.1 접속

- 백엔드를 `local` 또는 `dev` 프로필로 실행
- 브라우저에서 `/admin` 접속

### 6.2 서비스 탭

- `서비스` 탭에서 JSON 파일 1개 선택
- 업로드 대상 API:
  - `POST /api/v1/dev/admin/imports/service-catalog`
- multipart 필드명:
  - `file`
- 같은 탭에서 서비스 / 요금제 조회 가능

### 6.3 프로모션 탭

- `프로모션` 탭에서 JSON 파일 1개 이상 선택
- 업로드 대상 API:
  - `POST /api/v1/dev/admin/imports/promotions`
- multipart 필드명:
  - `files`
- 같은 탭에서 프로모션 조회 가능

### 6.4 결제내역 탭

- `결제내역` 탭에서 `userId`와 JSON 파일을 함께 입력 후 import
- 업로드 대상 API:
  - `POST /api/v1/dev/admin/payment-histories?userId={id}`
- multipart 필드명:
  - `file`
- 같은 탭에서 `userId` 기준 결제내역 조회 가능

### 6.5 수동 알림 탭

- `수동 알림` 탭에서 사용자와 메시지 정보를 입력
- 최신 FCM 토큰이 있으면 즉시 발송

### 6.6 배치 실행 탭

- `배치 실행` 탭에서 원하는 Job 버튼 클릭
- CHECKIN / PROMO 대상 선정과 발송을 각각 직접 실행 가능

## 7. JSON 파일 입력 규칙

### 7.1 서비스 / 요금제 JSON

- 서비스와 요금제 정보를 한 파일에서 함께 받음
- 서비스 참조는 `serviceCode` 기준

### 7.2 프로모션 JSON

- 프로모션은 여러 파일 업로드 허용
- 서비스 참조는 `services[].serviceCode` 기준
- 이미지 필드명은 `imageUrl`
- 날짜 필드는 offset 없는 `LocalDateTime` 문자열 사용
  - 예: `2026-03-22T22:21:55.268463`

### 7.3 결제내역 JSON

- 루트는 배열 `[]`
- 각 원소는 아래 필드를 포함
  - `paymentId`
  - `merchantRaw`
  - `amount`
  - `paymentDate`
  - `category`
- `paymentDate`는 `yyyy-MM-dd` 형식 사용

## 8. 후순위

- 관리자 화면에서 기준 데이터 직접 수정하는 기능

## 9. 결론

현재 관리자 웹은 데모 운영에 필요한 최소 제어 도구로 본다.

핵심 역할은 아래 5가지다.

1. 서비스 / 요금제 import 및 조회
2. 프로모션 import 및 조회
3. 결제내역 import 및 조회
4. 수동 알림 발송
5. CHECKIN / PROMO 배치 수동 실행
