# AI 폴더 구조 문서

OTT 구독 최적화 서비스를 위한 AI 파트 작업 폴더입니다.
크게 **데이터 생성**, **온디바이스 AI**, **스크래핑 자동화** 세 가지 영역으로 구성됩니다.

---

## 폴더 구조

```
AI/
├── docs/
├── datagenerator/
│   └── rawdata/          # .gitignore 처리
├── ondevice/
│   └── FE/
└── scraping/
    ├── card/
    ├── dbgenerator/
    ├── llm/
    ├── ott/
    ├── scheduler/
    └── telecom/
```

---

## 폴더별 설명

### 📁 docs
AI 파트 작업 내용을 문서화하는 폴더입니다.

설계 결정 이유, 폴더 구성 설명, 금일 작업 내용을 기록합니다.
팀원 간 컨텍스트 공유 및 인수인계(특히, FE)를 위해 작업 진행 중 지속적으로 업데이트합니다.

---

### 📁 datagenerator
사용자 결제 데이터 더미 생성 작업 폴더입니다.

실제 결제 데이터를 기반으로 테스트용 더미 데이터를 생성하는 코드를 관리합니다.
온디바이스 AI가 결제 내역을 정규화하는 기능을 테스트하기 위한 샘플 데이터 제공이 목적입니다.

- **rawdata/** : 실제 결제 데이터 원본 보관 폴더. 개인정보 보호를 위해 `.gitignore` 처리되어 git에 업로드되지 않습니다.

---

### 📁 ondevice
사용자 기기에서 결제 데이터를 분류·정규화하는 온디바이스 AI 이식 작업 폴더입니다.

결제 데이터를 서버로 전송하지 않고 기기 내에서 처리하여 개인정보를 보호하는 것이 핵심 목적입니다.
LLM을 모바일 기기에 이식하여 구독 서비스명, 결제 금액, 결제 주기 등을 추출합니다.

- **FE/** : 실제 프론트엔드 프로젝트 폴더를 가져와 작업합니다. 충돌 방지를 위해 AI 레포에서 별도로 작업한 후 매일 실제 FE 브랜치로 머지하여 충돌 여부를 확인합니다.

---

### 📁 scraping
OTT 요금제, 카드 혜택, 통신사 할인 등 구독 관련 데이터를 자동 수집하는 스크래핑 자동화 폴더입니다.

각 도메인(OTT, 카드, 통신사)을 독립적인 클래스로 구현하여 유지보수와 확장이 용이한 구조로 설계하였습니다.
수집된 데이터는 LLM을 통해 백엔드 DB 스키마에 맞게 정규화한 후 저장합니다.

- **ott/** : OTT 공식 홈페이지 스크래핑 코드. OTT별로 클래스를 분리하여 관리합니다. (티빙, 넷플릭스, 디즈니+ 등)
- **card/** : 카드사별 OTT 관련 혜택 스크래핑 코드. 카드별로 클래스를 분리하여 관리합니다.
- **telecom/** : 통신사 공홈 및 구독 마켓 스크래핑 코드. 통신사별로 클래스를 분리하여 관리합니다. (KT, LGU+ 등)
- **llm/** : 수집된 비정형 텍스트를 백엔드 DB 스키마에 맞게 정규화하는 LLM 관련 공통 모듈.
- **dbgenerator/** : 정규화된 데이터를 DB에 저장하는 코드.
- **scheduler/** : ott, card, telecom 클래스를 주기적으로 실행하는 스케줄러. 전체 스크래핑 파이프라인을 통합 관리합니다.

---

## 스크래핑 파이프라인 흐름

```
scheduler
    ↓
ott                    (각 도메인 스크래핑, 이후 card / telecom 순차 확장)
    ↓
llm                    (비정형 텍스트 → DB 스키마 정규화)
    ↓
dbgenerator            (MySQL 저장)
```

---

## 주의사항

- `datagenerator/rawdata/` 는 개인정보가 포함된 실제 결제 데이터가 있으므로 `.gitignore`에 등록되어 있습니다. 해당 폴더는 절대 git에 업로드하지 않습니다.
- 환경변수(API Key 등)는 `.env` 파일로 관리하며 git에 업로드하지 않습니다.
