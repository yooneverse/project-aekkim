DB_HOST = "mysql" 로 설정하는 이유 — docker-compose는 모든 서비스를 같은 브릿지 네트워크에 연결하고, 컨테이너명을 그대로 hostname 으로 등록합니다. 그래서 scraper 컨테이너 안에서 "mysql" 에 TCP 연결을 시도하면 Docker 내부 DNS가 mysql 컨테이너의 실제 IP 주소로 자동 해석합니다. 외부 인터넷으로 나가지 않고, 컨테이너 내부 네트워크로만 통신하므로 3306 포트를 외부에 노출할 필요가 없습니다.

SECTION 04
SQL 저장 로직 상세
실제 실행되는 SQL과 PK / UNIQUE KEY 기준
① services 테이블
PK: code — 없으면 INSERT, 있으면 UPDATE
INSERT INTO services (
  code, name, category,
  logo_url, cancel_guide_url,
  customer_service_phone
)
ON DUPLICATE KEY UPDATE
  name     = VALUES(name),
  logo_url = VALUES(logo_url),
  ...
PK code — TVING / NETFLIX / WAVVE / DISNEY_PLUS / WATCHA
Python logo_url, cancel_guide_url → base.py 상수에서 주입 (LLM 아님)
② service_plans 테이블
복합 PK: (service_code + plan_name)
INSERT INTO service_plans (
  service_code, plan_name,
  billing_cycle, monthly_price
)
ON DUPLICATE KEY UPDATE
  billing_cycle = VALUES(billing_cycle),
  monthly_price = VALUES(monthly_price);
PK (service_code, plan_name) 복합 키
FK service_code → services.code
Python YEARLY: monthly_price = rawPrice ÷ 12 (Python 직접 계산)
③ promotions 테이블
UNIQUE KEY: (title + starts_at) — BUNDLE / CARD_BENEFIT / PROMO 모두 처리
INSERT INTO promotions (
  promotion_type, title, summary,
  original_price, discount_price,
  starts_at, ends_at,
  source_url, service_codes, collected_at
)
ON DUPLICATE KEY UPDATE
  summary        = VALUES(summary),
  original_price = VALUES(original_price),
  service_codes  = VALUES(service_codes),
  collected_at   = VALUES(collected_at);
UNIQUE (title, starts_at) — 같은 제목 + 시작일 = 동일 프로모션
Python BUNDLE의 starts_at = 수집 시각(KST), ends_at = 2099-12-31 고정
주의 BUNDLE은 실행마다 starts_at이 달라지므로 매번 새 행이 삽입됨 → Spring 팀과 확인 필요

SECTION 08
Spring DDL 대조 필요 항목 ⚠️
Spring DDL 받으면 반드시 확인 후 client.py 수정 필요
⚠️ 아래 3가지가 해결되지 않은 상태에서 서버 DB에 연결하면 SQL 오류가 발생합니다.
1
Service 컬럼 불일치
현재 client.py는 cancel_guide_url, customer_service_phone, contact_email 을 INSERT하지만, ERD에 이 컬럼이 없을 수 있음. Spring DDL 확인 후 컬럼 추가 또는 SQL 제거 필요.
2
ServicePlan FK 방식
현재는 service_code (VARCHAR) 로 INSERT하지만, ERD는 serviceId (BIGINT FK) 를 사용할 수 있음. 그럴 경우 serviceId를 먼저 조회 후 INSERT로 변경 필요.
3
Promotion 서비스 연결
현재는 service_codes JSON 컬럼에 배열로 저장하지만, ERD는 PromotionService 별도 테이블로 관계를 맺을 수 있음. 행 분리 INSERT로 변경 필요할 수 있음.
