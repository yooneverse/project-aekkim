# 온디바이스 AI 구독분석 호출 책임 분리

> 작성일: 2026-03-19
> 대상: FE / BE / AI
> 목적: 결제내역 기반 구독분석 기능에서 "온디바이스 AI" 조건을 유지할 때 AI 호출 주체와 시스템 책임을 명확히 정리한다.

## 1. 결정 사항

- 본 프로젝트의 구독분석 AI는 `온디바이스`를 고정 원칙으로 한다.
- 따라서 AI 모델은 `사용자 안드로이드 기기 내부`에 탑재되어야 한다.
- AI 추론 실행 주체는 `프론트엔드(Android 앱)`다.
- 백엔드는 결제내역 조회 API 제공, 분석 결과 검증, DB 반영을 담당한다.
- 백엔드가 AI 모델을 직접 호출해 결제내역을 분석하면 그것은 온디바이스가 아니라 `서버 AI`로 간주한다.

## 2. 핵심 결론

### 2-1. AI는 어디에 있어야 하는가

- 앱 번들 내부 모델 파일
- 또는 앱 설치 후 기기에 내려받아 저장한 모델 파일
- 또는 모바일 추론 런타임이 기기 내부에서 실행하는 모델

즉, 모델은 서버가 아니라 `사용자 휴대폰 내부`에 존재해야 한다.

### 2-2. 누가 AI를 호출해야 하는가

- `온디바이스` 조건이면 FE가 호출해야 한다.
- BE는 AI를 호출하지 않는다.

### 2-3. 누가 DB를 반영해야 하는가

- DB 반영은 BE가 담당한다.
- FE는 추론 결과를 정해진 결과 형식(JSON 등)으로 BE에 전달한다.
- BE는 결과를 검증한 뒤 DB에 반영한다.

## 3. 역할 분리

### 3-1. 프론트엔드(FE)

- BE의 `GET /api/v1/payment-history` 응답으로 결제내역을 전달받는다.
- 기기 내 모델을 사용해 결제내역 + 로컬 데이터(예: UsageStats)를 분석한다.
- 분석 결과를 서버 계약에 맞는 구조화된 데이터로 변환한다.
- 분석 결과만 BE로 업로드한다.

### 3-2. 백엔드(BE)

- 저장된 결제내역을 `GET /api/v1/payment-history`로 제공한다.
- FE가 보낸 분석 결과를 검증한다.
- 검증된 결과를 DB에 저장하거나 갱신한다.
- 재처리, 중복 방지, 상태 관리, 로깅을 담당한다.

### 3-3. AI/Data

- 온디바이스 추론이 가능한 모델 형식과 입력/출력 계약을 정의한다.
- 앱 탑재 가능한 모델 크기, 정확도, 추론 시간 기준을 정한다.
- FE가 구현할 전처리/후처리 규칙을 문서화한다.

## 4. 가입 시 권장 데이터 흐름

1. 사용자가 가입 및 결제내역 연동을 완료한다.
2. FE가 `GET /api/v1/payment-history`로 결제내역을 조회한다.
3. BE는 저장된 결제내역을 응답으로 FE에 전달한다.
4. FE는 기기 내부 모델로 결제내역을 분석한다.
5. FE는 분석 결과를 정해진 결과 DTO 형식으로 BE에 전송한다.
6. BE는 결과를 검증하고 구독/결제일/분석 상태 등을 DB에 반영한다.
7. FE는 완료 상태를 조회하거나 다음 화면으로 이동한다.

## 5. 금지 패턴

다음 방식은 온디바이스 원칙과 충돌한다.

- BE가 OpenAI, Claude, 사내 AI 서버 등 외부 모델 API를 직접 호출해 결제내역을 분석하는 방식
- FE가 외부 서버 AI에 직접 결제내역 원문을 보내고 결과만 받는 방식
- 결제 원본 데이터를 서버 DB에 영구 저장한 뒤 서버 배치로 AI 분석하는 방식

위 방식들은 모두 `온디바이스 AI`가 아니라 `서버 AI` 또는 `하이브리드 서버 분석`으로 분류해야 한다.

## 6. 현재 프로젝트에 대한 해석

- 현재 BE는 결제내역 저장 및 `GET /api/v1/payment-history` 제공 책임을 가진다.
- 현재 FE는 UsageStats 등 기기 로컬 데이터를 읽는 구조를 이미 일부 갖고 있다.
- 따라서 온디바이스 고정 조건을 유지하려면:
  - BE가 결제내역 조회 API 제공
  - FE가 온디바이스 추론 수행
  - BE가 결과 저장
  구조로 가는 것이 현재 설계와 가장 일관된다.

## 7. 구현 시 필요한 계약

최소한 아래 계약이 필요하다.

- `GET /api/v1/payment-history`
- `온디바이스 분석 결과 업로드 API`
- `분석 상태/최종 결과 조회 API`
- AI 입력 스키마
- AI 출력 스키마
- 결과 검증 규칙

## 8. 한 줄 정리

`온디바이스 AI`가 고정이면, AI 모델은 사용자 기기에 탑재되고 AI 추론은 FE가 수행하며, BE는 `GET /api/v1/payment-history` 제공과 결과 저장을 담당한다.
