# 애낌 서비스 전체 구조

> 기준 일자: 2026-03-29
> 기준 범위: 현재 저장소 구조와 구현 상태 기준 현행화

## 1. 개요

애낌 프로젝트는 `BE`, `FE`, `AI`, `INFRA`, `docs`로 구성된 모노레포 구조다.
루트 `docs`는 서비스 정본 문서, `FE/docs/maintenance`는 프론트 운영 및 유지보수 기준 문서를 담당한다.

## 2. 저장소 구조

| 경로 | 역할 |
| --- | --- |
| `BE/` | Spring Boot 기반 백엔드 API 서버 |
| `FE/` | Android 앱(Kotlin, Jetpack Compose) |
| `AI/` | 스크래퍼 및 추천/정규화 보조 로직 |
| `INFRA/` | Docker Compose, Nginx, 배포 스크립트 |
| `docs/` | 요구사항, 화면명세, API, 구조 문서 |
| `seed/` | 시드 및 데이터 반영 리소스 |

## 3. 백엔드 구조

### 3.1 역할

백엔드는 인증, 사용자, 구독, 추천, 알림, 체크인, 결제 내역 관련 REST API를 제공한다.

### 3.2 핵심 구성

- `BE/src/main/java/com/ssafy/e106/domain/auth`
- `BE/src/main/java/com/ssafy/e106/domain/user`
- `BE/src/main/java/com/ssafy/e106/domain/subscription`
- `BE/src/main/java/com/ssafy/e106/domain/notification`
- `BE/src/main/java/com/ssafy/e106/domain/promotion`
- `BE/src/main/java/com/ssafy/e106/global`

### 3.3 현재 FE와 직접 연결되는 주요 API 범위

- 인증: Google, Kakao 로그인, 토큰 갱신, 로그아웃
- 사용자: 내 정보 조회, 알림 설정, 마케팅 수신 동의, 회원탈퇴
- 구독: 목록 조회, 상세 조회, 생성, 수정, 삭제
- 서비스/번들: 서비스 목록, 요금제 목록, 번들 목록
- 추천: 추천 목록, 추천 상세
- 체크인: 체크인 조회, 응답 제출
- 알림: FCM 토큰 등록, 알림 조회, 읽음 처리, 삭제

## 5. AI 구조

### 5.1 역할

`AI/`는 서비스/프로모션 데이터 수집과 정규화 보조 역할을 담당한다.
현재 Android 앱의 화면 흐름을 직접 제어하지는 않지만, 추천 및 데이터 적재 기반을 지원한다.

### 5.2 핵심 경로

| 경로 | 역할 |
| --- | --- |
| `AI/scraper/ott` | OTT 관련 정보 수집 |
| `AI/scraper/promotion` | 프로모션 정보 수집 |
| `AI/scraper/llm` | 정규화 및 프롬프트 관련 로직 |
| `AI/scraper/db` | DB 반영 로직 |
| `AI/scheduler.py` | 스케줄 실행 진입점 |

## 6. 인프라 구조

### 6.1 핵심 파일

| 경로 | 역할 |
| --- | --- |
| `INFRA/docker-compose.dev.yml` | 개발 환경 오케스트레이션 |
| `INFRA/docker-compose.prod.yml` | 운영 환경 오케스트레이션 |
| `INFRA/nginx/e106.conf` | Nginx 라우팅 설정 |
| `INFRA/deploy-dev.sh` | 개발 배포 스크립트 |
| `INFRA/deploy-prod.sh` | 운영 배포 스크립트 |
| `Jenkinsfile` | CI/CD 파이프라인 |

### 6.2 운영 구성 요약

- 앱은 백엔드 API와 통신한다.
- 백엔드는 인증, 구독, 추천, 알림 데이터를 제공한다.
- AI 수집 결과는 백엔드/DB 운영 데이터의 기반이 된다.
- Nginx와 Docker Compose 기반으로 배포 환경이 구성된다.

## 7. 현재 사용자 흐름 관점의 시스템 연결

1. 사용자가 Android 앱에서 로그인한다.
2. 앱은 인증 API로 세션을 만든다.
3. 앱은 사용자/구독/추천/알림 API를 호출해 화면을 구성한다.
4. 알림은 FCM 토큰 등록을 통해 앱과 연결된다.
5. 추천/카탈로그 계열 데이터는 AI 및 시드 기반 데이터와 연결된다.

## 8. 문서 구조 기준

### 8.1 루트 문서

- 요구사항 정의서
- 화면명세서
- 서비스 전체 구조
- API 문서
- 변경사항 정리 문서

### 8.2 FE 문서

- `FE/docs/maintenance`: 운영, 유지보수, 구현 기준
- `FE/docs/archive`: 계획서, 참고 문서, 이력성 문서

## 9. 현행 기준 주의사항

1. 실제 앱 기준은 `FE` 코드와 `FE/docs/maintenance`를 우선으로 본다.
2. 루트 `docs`는 대외 공유용 정본 성격을 유지한다.
3. 더 이상 사용하지 않는 기획 초안이나 과거 화면 설명은 정본으로 간주하지 않는다.
