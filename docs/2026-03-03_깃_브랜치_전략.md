## 브랜치 전략 (Git Flow 기반)

```
main (production)
├── develop (개발 통합)
│   ├── be/feat/S14P21E106-42-user-login
│   ├── be/fix/S14P21E106-57-null-pointer
│   ├── be/refactor/S14P21E106-63-cleanup
│   ├── fe/feat/S14P21E106-12-login-page
│   ├── fe/fix/S14P21E106-23-button-event
│   └── fe/refactor/S14P21E106-30-styling
└── hotfix/S14P21E106-99-auth-fix
```

- **`{파트}/{타입}/{번호}-{설명}`** 형식
- 번호: Jira Task 이슈 번호 (예: `S14P21E106-42`)
- **소문자 + 하이픈** (kebab-case)

```
be/feat/S14P21E106-42-user-login
be/fix/S14P21E106-57-null-pointer-exception
fe/feat/S14P21E106-12-login-page
be/refactor/S14P21E106-63-user-service-cleanup
```
