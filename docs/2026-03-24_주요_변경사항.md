## 2. 이번 정리에서 반영한 핵심 기준

### 2.1 로그인/온보딩

- Google 로그인과 Kakao 로그인은 실제 구현 기준으로 문서화했다.
- Naver 로그인은 버튼 노출만 존재하고 실제 인증은 미구현 상태로 정리했다.
- 약관 상세는 외부 링크가 아니라 앱 내부 상세 화면 기준으로 수정했다.

### 2.2 메인 화면

- 기본 진입 화면은 세션 유무에 따라 온보딩 또는 대시보드로 분기되는 구조로 반영했다.
- 대시보드는 월 총액, 절감 가능 금액, 리마인드 배너, 검토 필요 배너, 구독 목록, 알림 진입 구조로 정리했다.
- 수동 구독 추가는 다단계 바텀시트 흐름 기준으로 정리했다.
- 검토 필요 결제 내역은 별도 수동 매핑 화면으로 구분해 반영했다.

### 2.3 인사이트/추천

- 인사이트 화면은 히트맵형 설명을 제거하고 현재 구현된 요약 카드, TOP 3, 구독별 목록 구조로 수정했다.
- 추천 화면은 카테고리 탭과 추천 유형 탭(번들, 프로모션, 카드 혜택) 기준으로 다시 정리했다.

### 2.4 체크인/알림

- 체크인, 후속 제안, 해지 가이드 흐름을 현재 구현에 맞게 정리했다.
- 알림함을 별도 화면으로 반영했고, 개별 삭제 및 전체 삭제 기능을 명시했다.

### 2.5 설정/계정

- 마이페이지의 알림 설정, 마케팅 수신 동의, 약관 확인, 로그아웃 구조를 최신화했다.
- 회원탈퇴는 단순 다이얼로그가 아니라 전용 단계형 화면으로 수정 반영했다.

## 3. 문서 정리 원칙

1. 실제 FE 라우트와 화면이 우선이다.
2. 구현되지 않은 확장 기능은 별도 차기 항목으로만 남긴다.
3. 운영 기준은 `FE/docs/maintenance`, 대외 정본은 루트 `docs`로 역할을 구분한다.

## 4. 남아 있는 후속 과제

1. Naver 실제 로그인 연동 여부 확정
2. 문의/고객지원 화면 추가 여부 결정
3. 인사이트 시각화 고도화 여부 검토
4. 추천 및 알림 개인화 세분화
