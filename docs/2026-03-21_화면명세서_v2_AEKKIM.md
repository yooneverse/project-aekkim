# 애낌 (AEKKIM) 화면명세서

> 문서 버전: v2.6
> 기준 일자: 2026-03-29
> 기준 범위: 현재 구현된 Android 앱 화면 기준 현행화

## 1. 문서 목적

이 문서는 현재 앱에 실제로 존재하는 화면과 라우트를 기준으로 화면 구성과 동작을 정리한 문서다.
기존 기획 표현 중 현행 구현과 다른 내용은 제외하고, 실제 사용자 흐름에 맞춰 정리했다.

## 2. 전역 내비게이션

### 2.1 시작 조건

- 세션 없음: `AuthRoute.Onboarding`
- 세션 있음: `HomeRoute.Dashboard`

### 2.2 하단 탭

| 탭 라벨 | 라우트 | 역할 |
| --- | --- | --- |
| 구독 | `HomeRoute.Dashboard` | 구독 현황, 수동 추가, 상세 진입 |
| 인사이트 | `HomeRoute.Insight` | 최근 사용량 기반 인사이트 |
| 프로모션 | `HomeRoute.PromotionList` | 카테고리별 추천 목록 |
| MY | `HomeRoute.MyPage` | 설정, 약관, 로그아웃, 탈퇴 |

### 2.3 주요 딥링크

- `aekkim://subscriptions/{id}`
- `aekkim://promotions/{id}`
- `aekkim://checkin/{subscriptionId}?isFullScreen=...`
- `aekkim://cancel/{id}`
- `aekkim://legal/service-terms`
- `aekkim://legal/privacy-policy`
- `aekkim://legal/marketing-consent`

## 4. 화면 상세

### SCR-001 온보딩/로그인

| 항목 | 내용 |
| --- | --- |
| 목적 | 첫 진입 사용자가 앱 가치를 이해하고 로그인 시작 |
| 주요 요소 | 타이틀, 소개 문구, Google 로그인 버튼, Kakao 로그인 버튼, Naver 로그인 버튼 |
| 주요 액션 | Google 로그인, Kakao 로그인, Naver 안내 토스트 |
| 비고 | Naver는 실제 로그인 미지원, 안내 토스트만 제공 |

### SCR-001-2 약관 동의 및 권한

| 항목 | 내용 |
| --- | --- |
| 목적 | 서비스 이용 전 필수 동의와 권한 흐름 완료 |
| 주요 요소 | 전체 동의, 개별 동의, 약관 상세 진입, 알림 권한 안내, Usage Stats 권한 안내 |
| 주요 액션 | 필수 약관 체크, 선택 동의 체크, 알림 권한 요청 또는 건너뛰기, Usage Stats 설정 이동 |
| 비고 | 약관 상세는 외부 링크가 아니라 앱 내부 문서 화면으로 이동 |

### SCR-001-3 약관 상세 화면

| 항목 | 내용 |
| --- | --- |
| 목적 | 서비스 이용약관, 개인정보 처리방침, 마케팅 동의 내용 확인 |
| 주요 요소 | 문서 제목, 본문, 상단 뒤로가기 |
| 주요 액션 | 문서 확인 후 이전 화면 복귀 |
| 비고 | 인증 전/후 모두 동일한 내부 화면 구조 사용 |

## 5. 화면 간 주요 흐름

### 5.1 신규 사용자

1. 온보딩/로그인
2. 약관 동의 및 권한
3. 분석 로딩
4. 구독 확인
5. 대시보드

### 5.2 기존 사용자

1. 앱 시작
2. 대시보드
3. 하단 탭 또는 알림/딥링크를 통한 화면 이동

### 5.3 체크인 관련 흐름

1. 알림 또는 내부 진입
2. 체크인
3. 필요 시 후속 제안
4. 해지 가이드 또는 대시보드 복귀

### 5.4 설정 관련 흐름

1. MY 탭 진입
2. 설정 변경 또는 문서 확인
3. 로그아웃 또는 회원탈퇴

## 6. 현행 기준 유의사항

1. Naver 로그인은 화면에 존재하지만 실제 인증은 제공하지 않는다.
2. 약관 상세는 외부 Notion 또는 웹 링크가 아니라 내부 화면이다.
3. 인사이트 화면은 히트맵형 구조가 아니라 요약 카드와 목록형 구조다.
4. 알림함은 별도 화면으로 존재하며 개별 삭제와 전체 삭제를 지원한다.
5. 회원탈퇴는 단순 다이얼로그가 아니라 전용 단계형 화면이다.
