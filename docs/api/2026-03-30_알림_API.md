# 알림 API

## 공통 규칙

- Base URL: `/api/v1/notifications`
- 공통 응답 형식: `ApiResponse<T>`
- 모든 API는 `Authorization: Bearer {accessToken}` 헤더가 필요하다.

---

## 1. FCM 토큰 등록/갱신

- Method: `POST`
- URL: `/api/v1/notifications/tokens`
- 설명: 현재 사용자의 FCM 토큰을 등록하거나 갱신한다.

### Request Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `fcmToken` | string | Y | FCM 등록 토큰 |

### Response Data

| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| `fcmTokenId` | long | FCM 토큰 ID |
| `fcmToken` | string | 저장된 FCM 토큰 |
| `updatedAt` | string(datetime) | 갱신 시각 |

---

## 2. 알림 목록 조회

- Method: `GET`
- URL: `/api/v1/notifications`
- 설명: 현재 사용자의 알림 목록을 조회한다. 실제 발송 완료된 알림만 응답에 포함한다.

### Query Parameter

| 변수명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `unreadOnly` | boolean | N | 미읽음 알림만 조회 여부, 기본값 `false` |
| `size` | int | N | 조회 건수, 기본값 `20`, 최대 `100` |

### Response Data

| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| `notifications` | array(object) | 알림 목록 |
| `notifications[].notificationId` | long | 알림 ID |
| `notifications[].type` | string | 알림 유형 (`CHECKIN`, `CHURN_REVIEW`, `PROMO`) |
| `notifications[].referenceId` | long | 참조 ID |
| `notifications[].title` | string | 제목 |
| `notifications[].body` | string | 본문 |
| `notifications[].isRead` | boolean | 읽음 여부 |
| `notifications[].readAt` | string(datetime) | 읽음 시각 |
| `notifications[].sentAt` | string(datetime) | 발송 시각 |

---

## 3. 알림 읽음 처리

- Method: `PATCH`
- URL: `/api/v1/notifications/{notificationId}/read`
- 설명: 특정 알림을 읽음 상태로 변경한다.

### Path Variable

| 변수명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `notificationId` | long | Y | 알림 ID |

### Response Data

| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| `notificationId` | long | 알림 ID |
| `isRead` | boolean | 읽음 여부 |
| `readAt` | string(datetime) | 읽음 시각 |

### Error

| HTTP Status | code | 발생 조건 |
| --- | --- | --- |
| `401` | `UNAUTHORIZED` | 액세스 토큰 누락, 만료, 위조 |
| `403` | `NOTIFICATION_FORBIDDEN` | 타인 알림 읽음 처리 시도 |
| `404` | `NOTIFICATION_NOT_FOUND` | `notificationId`가 유효하지 않음 |

---

## 4. 알림함 전체 삭제

- Method: `DELETE`
- URL: `/api/v1/notifications`
- 설명: 현재 사용자의 발송 완료 알림을 전체 삭제한다.

### Response

- HTTP Status: `200 OK`
- `data = null`

### Success Example

```json
{
  "success": true,
  "data": null,
  "message": "알림함을 비웠습니다."
}
```

---

## 5. 알림 삭제

- Method: `DELETE`
- URL: `/api/v1/notifications/{notificationId}`
- 설명: 특정 알림 하나를 삭제한다.

### Path Variable

| 변수명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `notificationId` | long | Y | 알림 ID |

### Response

- HTTP Status: `200 OK`
- `data = null`

### Error

| HTTP Status | code | 발생 조건 |
| --- | --- | --- |
| `401` | `UNAUTHORIZED` | 액세스 토큰 누락, 만료, 위조 |
| `403` | `NOTIFICATION_FORBIDDEN` | 타인 알림 삭제 시도 |
| `404` | `NOTIFICATION_NOT_FOUND` | `notificationId`가 유효하지 않거나 아직 발송되지 않은 알림 |
