# 사용자 API

## 공통 규칙

- Base URL: `/api/v1/users`
- 공통 응답 형식: `ApiResponse<T>`
- 모든 API는 `Authorization: Bearer {accessToken}` 헤더가 필요하다.

---

## 1. 내 정보 조회

- Method: `GET`
- URL: `/api/v1/users/me`
- 설명: 현재 로그인한 사용자의 기본 정보와 알림 설정 상태를 조회한다.

### Response Data

| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| `userId` | long | 사용자 ID |
| `displayName` | string | 사용자 표시 이름 |
| `email` | string | 사용자 이메일 |
| `profileImageUrl` | string | 프로필 이미지 URL |
| `checkinAlertEnabled` | boolean | 체크인 알림 수신 여부 |
| `promoAlertEnabled` | boolean | 프로모션 알림 수신 여부 |
| `optionalConsentAgreed` | boolean | 선택 동의 여부 |
| `lastLoginAt` | string(datetime) | 최근 로그인 시각 |

### Success Example

```json
{
  "success": true,
  "data": {
    "userId": 12,
    "displayName": "애낌유저",
    "email": "user@example.com",
    "profileImageUrl": "https://cdn.example.com/profile.png",
    "checkinAlertEnabled": true,
    "promoAlertEnabled": true,
    "optionalConsentAgreed": false,
    "lastLoginAt": "2026-03-30T09:10:21"
  },
  "message": "내 정보 조회에 성공했습니다."
}
```

## 2. 회원 탈퇴

- Method: `DELETE`
- URL: `/api/v1/users/me`
- 설명: 현재 로그인한 사용자를 삭제한다.

### Response

- HTTP Status: `200 OK`
- `data = null`

### Success Example

```json
{
  "success": true,
  "data": null,
  "message": "회원 탈퇴에 성공했습니다."
}
```

### 비고

- 체크인 이력, 구독, 알림, FCM 토큰은 서비스 로직에서 함께 삭제한다.
- Redis refresh token도 함께 제거한다.

---

## 3. 알림 설정 변경

- Method: `PATCH`
- URL: `/api/v1/users/me/notification-settings`
- 설명: 체크인 알림과 프로모션 알림 수신 여부를 변경한다.

### Request Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `checkinAlertEnabled` | boolean | Y | 체크인 알림 수신 여부 |
| `promoAlertEnabled` | boolean | Y | 프로모션 알림 수신 여부 |

### Response Data

| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| `checkinAlertEnabled` | boolean | 변경된 체크인 알림 수신 여부 |
| `promoAlertEnabled` | boolean | 변경된 프로모션 알림 수신 여부 |

### Success Example

```json
{
  "success": true,
  "data": {
    "checkinAlertEnabled": true,
    "promoAlertEnabled": false
  },
  "message": "알림 설정이 변경되었습니다."
}
```

## 4. 선택 사항 동의 변경

- Method: `PATCH`
- URL: `/api/v1/users/me/optional-consent`
- 설명: 사용자의 선택 사항 동의 여부를 변경한다.

### Request Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `optionalConsentAgreed` | boolean | Y | 선택 사항 동의 여부 |

### Response Data

| 필드명 | 타입 | 설명 |
| --- | --- | --- |
| `optionalConsentAgreed` | boolean | 변경된 선택 사항 동의 여부 |

### Success Example

```json
{
  "success": true,
  "data": {
    "optionalConsentAgreed": true
  },
  "message": "선택 사항 동의가 변경되었습니다."
}
```
