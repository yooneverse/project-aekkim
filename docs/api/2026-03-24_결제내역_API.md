# 결제내역 API

## 공통 사항

- Base URL: `/api/v1`
- 응답 포맷: `ApiResponse<T>`
- 결제내역 API는 회원가입/로그인 전에도 호출할 수 있습니다.
- `Authorization: Bearer {accessToken}` 헤더는 선택 사항입니다.
- 결제내역은 사용자 ID 없이 공용 결제내역 저장소 기준으로 저장·조회합니다.

---

## 1. 결제내역 저장

- Method: `POST`
- URL: `/api/v1/payment-history`
- 설명: 결제내역 목록을 저장합니다. 중복된 `paymentId`는 저장하지 않습니다.

### Request

#### Header

| 헤더명 | 필수 | 설명 |
| --- | --- | --- |
| `Authorization` | N | `Bearer {accessToken}` |
| `Content-Type` | Y | `application/json` |

#### Body

배열(`[]`) 형태의 JSON을 요청 본문으로 사용합니다.

| 필드 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `paymentId` | string | Y | 결제 식별자. 최대 100자 |
| `merchantRaw` | string | Y | 가맹점 원문명. 최대 255자 |
| `amount` | int | Y | 결제 금액. 0 초과 |
| `paymentDate` | string(date) | Y | 결제일. `yyyy-MM-dd` |
| `category` | string | N | 결제 카테고리. 최대 50자 |

#### Request Example

```json
[
  {
    "paymentId": "PAY-20260324-0001",
    "merchantRaw": "NETFLIX",
    "amount": 17000,
    "paymentDate": "2026-03-24",
    "category": "OTT"
  },
  {
    "paymentId": "PAY-20260324-0002",
    "merchantRaw": "TVING",
    "amount": 13500,
    "paymentDate": "2026-03-24",
    "category": "OTT"
  }
]
```

### Response

#### Success

- HTTP Status: `201 Created`

#### Data Fields

| 필드 | 타입 | Null 허용 | 설명 |
| --- | --- | --- | --- |
| `requestedCount` | int | N | 요청한 결제내역 개수 |
| `savedCount` | int | N | 실제 저장된 결제내역 개수 |
| `skippedCount` | int | N | 중복 등으로 건너뛴 개수 |
| `savedPayments` | array(object) | N | 저장된 결제내역 목록 |
| `savedPayments[].paymentHistoryId` | long | N | 결제내역 PK |
| `savedPayments[].paymentId` | string | N | 결제 식별자 |
| `savedPayments[].merchantRaw` | string | N | 가맹점 원문명 |
| `savedPayments[].amount` | int | N | 결제 금액 |
| `savedPayments[].paymentDate` | string(date) | N | 결제일 |
| `savedPayments[].category` | string | Y | 결제 카테고리 |
| `savedPayments[].createdAt` | string(datetime) | N | 생성 시각 |
| `savedPayments[].updatedAt` | string(datetime) | N | 수정 시각 |

#### Success Example

```json
{
  "success": true,
  "data": {
    "requestedCount": 2,
    "savedCount": 2,
    "skippedCount": 0,
    "savedPayments": [
      {
        "paymentHistoryId": 1,
        "paymentId": "PAY-20260324-0001",
        "merchantRaw": "NETFLIX",
        "amount": 17000,
        "paymentDate": "2026-03-24",
        "category": "OTT",
        "createdAt": "2026-03-24T23:30:00",
        "updatedAt": "2026-03-24T23:30:00"
      }
    ]
  },
  "message": "결제내역이 저장되었습니다."
}
```

## 2. 결제내역 조회

- Method: `GET`
- URL: `/api/v1/payment-history`
- 설명: 결제내역 저장소에 저장된 결제내역을 조회합니다.

### Request

#### Header

| 헤더명 | 필수 | 설명 |
| --- | --- | --- |
| `Authorization` | N | `Bearer {accessToken}` |

### Response

#### Success

- HTTP Status: `200 OK`

#### Data Fields

| 필드 | 타입 | Null 허용 | 설명 |
| --- | --- | --- | --- |
| `count` | int | N | 조회된 결제내역 개수 |
| `payments` | array(object) | N | 결제내역 목록 |
| `payments[].paymentHistoryId` | long | N | 결제내역 PK |
| `payments[].paymentId` | string | N | 결제 식별자 |
| `payments[].merchantRaw` | string | N | 가맹점 원문명 |
| `payments[].amount` | int | N | 결제 금액 |
| `payments[].paymentDate` | string(date) | N | 결제일 |
| `payments[].category` | string | Y | 결제 카테고리 |
| `payments[].createdAt` | string(datetime) | N | 생성 시각 |
| `payments[].updatedAt` | string(datetime) | N | 수정 시각 |

#### Success Example

```json
{
  "success": true,
  "data": {
    "count": 2,
    "payments": [
      {
        "paymentHistoryId": 1,
        "paymentId": "PAY-20260324-0001",
        "merchantRaw": "NETFLIX",
        "amount": 17000,
        "paymentDate": "2026-03-24",
        "category": "OTT",
        "createdAt": "2026-03-24T23:30:00",
        "updatedAt": "2026-03-24T23:30:00"
      }
    ]
  },
  "message": "결제내역을 조회했습니다."
}
```

### Error

| HTTP Status | code | message | 설명 |
| --- | --- | --- | --- |
| `400` | `INVALID_INPUT` | 잘못된 요청입니다. | 요청 형식/값 오류 |
| `500` | `INTERNAL_ERROR` | 서버 처리 중 오류가 발생했습니다. | 서버 내부 오류 |
