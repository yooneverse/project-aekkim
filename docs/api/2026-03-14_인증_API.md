# 인증 API

## 공통 규칙

- Base URL: `/api/v1/auth`
- 공통 응답 형식: `ApiResponse<T>`
- 로그인, 액세스 토큰 재발급 API는 `Authorization` 헤더가 필요 없다.
- 로그아웃 API는 `Authorization: Bearer {accessToken}` 헤더가 필요하다.

---

## 1. 구글 로그인

- Method: `POST`
- URL: `/api/v1/auth/login/google`
- 설명: Google ID Token으로 로그인하고 우리 서비스의 JWT를 발급한다.

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| `Content-Type` | Y | `application/json` |

참고:
- 로그인 API는 `Authorization` 헤더가 필요 없다.

#### Path Variable

없음

#### Query Parameter

없음

#### Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `googleIdToken` | string | Y | 구글 사용자 ID 토큰 |

#### Request Example

```json
{
  "googleIdToken": "google_id_token_value"
}
```

### Response

#### Success

- HTTP Status: `200 OK`

#### Data Fields

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| `accessToken` | string | N | 우리 서비스 JWT |
| `refreshToken` | string | N | 리프레시 토큰. Redis 관리 |
| `tokenType` | string | N | `Bearer` |
| `expiresIn` | long | N | 만료까지 남은 초(second) |
| `user.userId` | long | N | 사용자 ID (`users.user_id`) |
| `user.displayName` | string | N | 소셜 프로필 이름 (`users.display_name`) |
| `user.email` | string | Y | 소셜 이메일 (`users.email`) |
| `user.profileImageUrl` | string | Y | 소셜 프로필 이미지 URL (`users.profile_image_url`) |
| `user.checkinAlertEnabled` | boolean | N | 체크인 알림 수신 여부 (`users.checkin_alert_enabled`) |
| `user.promoAlertEnabled` | boolean | N | 프로모션 알림 수신 여부 (`users.promo_alert_enabled`) |
| `user.optionalConsentAgreed` | boolean | N | 선택 사항 동의 여부 (`users.optional_consent_agreed`) |

#### Success Example

```json
{
  "success": true,
  "data": {
    "accessToken": "jwt_access_token",
    "refreshToken": "jwt_refresh_token",
    "tokenType": "Bearer",
    "expiresIn": 1209600,
    "user": {
      "userId": 12,
      "displayName": "이동장",
      "email": "leedongjang@gmail.com",
      "profileImageUrl": "https://lh3.googleusercontent.com/a/ACg8ocK...",
      "checkinAlertEnabled": true,
      "promoAlertEnabled": true,
      "optionalConsentAgreed": false
    }
  },
  "message": "로그인에 성공했습니다."
}
```

### Error

| HTTP Status | code | message | 발생 조건 |
| --- | --- | --- | --- |
| `400` | `INVALID_INPUT` | 잘못된 입력입니다. | 토큰 값 누락 또는 형식 오류 |
| `401` | `AUTH_GOOGLE_TOKEN_INVALID` | 구글 토큰이 유효하지 않습니다. | 구글 토큰 검증 실패 |
| `500` | `INTERNAL_ERROR` | 서버 내부 오류가 발생했습니다. | 서버 예외 |

#### Error Example

```json
{
  "success": false,
  "code": "AUTH_GOOGLE_TOKEN_INVALID",
  "message": "구글 토큰이 유효하지 않습니다."
}
```

### 비고

- `refresh:{userId}` 키로 `refreshToken`을 Redis에 저장한다.
- 저장 시 TTL을 설정한다.

## 3. 로그아웃

- Method: `POST`
- URL: `/api/v1/auth/logout`
- 설명: 현재 사용자의 리프레시 토큰을 무효화한다.

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| `Authorization` | Y | `Bearer {accessToken}` |

#### Path Variable

없음

#### Query Parameter

없음

#### Body

없음

### Response

#### Success

- HTTP Status: `200 OK`

#### Data Fields

없음 (`data = null`)

#### Success Example

```json
{
  "success": true,
  "data": null,
  "message": "로그아웃에 성공했습니다."
}
```

### Error

| HTTP Status | code | message | 발생 조건 |
| --- | --- | --- | --- |
| `401` | `UNAUTHORIZED` | 인증이 필요합니다. | 액세스 토큰 누락, 만료, 위조 |
| `500` | `INTERNAL_ERROR` | 서버 내부 오류가 발생했습니다. | 서버 예외 |

#### Error Example

```json
{
  "success": false,
  "code": "UNAUTHORIZED",
  "message": "인증이 필요합니다."
}
```

### 비고

- `accessToken`에서 `userId`를 추출한다.
- Redis의 `refresh:{userId}` 키를 삭제한다.

## 4. 액세스 토큰 재발급

- Method: `POST`
- URL: `/api/v1/auth/refresh`
- 설명: 유효한 리프레시 토큰으로 액세스 토큰과 리프레시 토큰을 재발급한다.

### Request

#### Header

| 이름 | 필수 | 설명 |
| --- | --- | --- |
| `Content-Type` | Y | `application/json` |

참고:
- 재발급 API는 `Authorization` 헤더가 필요 없다.

#### Path Variable

없음

#### Query Parameter

없음

#### Body

| 필드명 | 타입 | 필수 | 설명 |
| --- | --- | --- | --- |
| `refreshToken` | string | Y | 재발급용 리프레시 토큰 |

#### Request Example

```json
{
  "refreshToken": "refresh_token_value"
}
```

### Response

#### Success

- HTTP Status: `200 OK`

#### Data Fields

| 필드명 | 타입 | Null 가능 | 설명 |
| --- | --- | --- | --- |
| `accessToken` | string | N | 새로 발급된 액세스 토큰 |
| `refreshToken` | string | N | 새로 발급된 리프레시 토큰 |
| `tokenType` | string | N | `Bearer` |
| `expiresIn` | long | N | 만료까지 남은 초(second) |

#### Success Example

```json
{
  "success": true,
  "data": {
    "accessToken": "new_jwt_access_token",
    "refreshToken": "new_jwt_refresh_token",
    "tokenType": "Bearer",
    "expiresIn": 1209600
  },
  "message": "액세스 토큰이 재발급되었습니다."
}
```

### Error

| HTTP Status | code | message | 발생 조건 |
| --- | --- | --- | --- |
| `400` | `INVALID_INPUT` | 잘못된 입력입니다. | `refreshToken` 누락 또는 형식 오류 |
| `401` | `AUTH_REFRESH_TOKEN_INVALID` | 유효하지 않은 리프레시 토큰입니다. | 토큰 위조 또는 검증 실패 |
| `401` | `AUTH_REFRESH_TOKEN_EXPIRED` | 리프레시 토큰이 만료되었습니다. | 만료된 리프레시 토큰 |
| `500` | `INTERNAL_ERROR` | 서버 내부 오류가 발생했습니다. | 서버 예외 |

#### Error Example

```json
{
  "success": false,
  "code": "AUTH_REFRESH_TOKEN_INVALID",
  "message": "유효하지 않은 리프레시 토큰입니다."
}
```

### 비고

- Redis의 `refresh:{userId}` 키를 조회한다.
- 토큰 검증 후 기존 키를 삭제한다.
- 새 액세스 토큰과 새 리프레시 토큰을 발급한 뒤 다시 저장한다.
