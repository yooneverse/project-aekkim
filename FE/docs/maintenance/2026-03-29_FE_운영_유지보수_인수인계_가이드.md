# FE 운영 유지보수 인수인계 가이드

> 작성일: 2026-03-29
> 대상: `fe/` Android 앱 운영, 유지보수, 후속 인수인계 담당자

## 1. 목적

- 프로젝트 마무리 이후 `fe/` 모듈을 다시 실행, 점검, 수정할 때 필요한 최소 기준을 한 문서로 정리한다.
- 초기 개발 계획서가 아니라 현재 코드 기준의 운영 정보만 남긴다.

## 2. 현재 기준 요약

- 앱 타입: Jetpack Compose 기반 Android 앱
- 패키지/식별자: `com.ssafy.e106`
- 주요 실행 모듈: `fe/app`
- 최소 SDK / 타깃 SDK: `26` / `35`
- 인증: Google 로그인 중심, Kakao SDK 초기화 코드 존재
- 알림: Firebase Cloud Messaging + 딥링크 라우팅
- 토큰 저장: `EncryptedSharedPreferences`, 디버그 에뮬레이터에서는 plain `SharedPreferences` fallback 가능
- 네트워크 기준 URL 선택:
  - 에뮬레이터: `BASE_URL_EMULATOR` 또는 `BASE_URL_DEV`
  - 실기기 디버그: `BASE_URL_DEVICE`
  - 릴리즈: `BASE_URL_PROD`

## 3. 필수 준비물

| 항목 | 위치 | 설명 |
| --- | --- | --- |
| Android SDK 경로 | `fe/local.properties` | `sdk.dir` 설정 필요 |
| API 서버 주소 | `fe/local.properties` | `BASE_URL_DEV`, `BASE_URL_EMULATOR`, `BASE_URL_DEVICE`, `BASE_URL_PROD` |
| Firebase 설정 | `fe/app/google-services.json` | Git 제외 파일이라 별도 준비 필요 |
| Google 로그인 설정 | `fe/local.properties` 또는 `google-services.json` | `GOOGLE_WEB_CLIENT_ID` 또는 `default_web_client_id` 필요 |
| Kakao 설정 | `fe/local.properties` | `KAKAO_NATIVE_APP_KEY` 설정 시 SDK 초기화 가능 |
| 공용 debug 서명 | `fe/signing/debug.keystore` | Firebase SHA 고정 목적 |

## 4. 빠른 실행 절차

1. Android Studio에서 `fe/`를 연다.
2. `fe/local.properties`를 환경에 맞게 채운다.
3. `fe/app/google-services.json`을 배치한다.
4. 에뮬레이터 또는 실기기를 띄운다.
5. 아래 명령으로 빌드/설치한다.

```bash
.\gradlew.bat :app:installDebug
.\gradlew.bat :app:assembleDebug
.\gradlew.bat :app:assembleRelease
.\gradlew.bat :app:bundleRelease
```

## 6. 화면/기능 유지보수 기준

### 현재 운영 대상으로 보는 기능

- 온보딩, 약관 동의, 권한 요청
- 분석 로딩, 구독 확인
- 대시보드, 수동 추가/수정, 수동 확인
- 구독 상세, 체크인, 후속 권유, 해지 지원
- 프로모션 목록/상세
- 알림함, 딥링크 이동
- 마이페이지, 알림 설정, 선택 동의, 로그아웃, 회원탈퇴

### 유지보수 시 주의할 갭

- Google 로그인은 기준 흐름으로 유지하되, Kakao/Naver 정책은 다시 확인이 필요하다.
- `TextSecondary`, 탭바 indicator, 권한 다이얼로그 정책은 디자인 후속 과제가 남아 있다.
- 분석 초기 단계에서 `0원` 노출 가능성이 남아 있다.
- 표준 테스트 소스셋과 자동화 테스트는 사실상 비어 있으므로 회귀는 수동 검증 비중이 높다.
- 릴리즈 서명/실서버 URL/Analytics 운영 기준은 최종 확정 문서가 없다.

## 7. 외부 연동 의존성

| 영역 | 현재 코드 기준 | 운영 메모 |
| --- | --- | --- |
| Backend API | Retrofit + OkHttp | URL 오설정 시 대부분 기능이 동시에 실패한다. |
| Google 로그인 | Credential Manager + Google ID | `google-services.json`과 클라이언트 ID 설정을 함께 본다. |
| Kakao 로그인 | SDK 초기화 코드 존재 | `KAKAO_NATIVE_APP_KEY`가 비어 있으면 초기화 자체를 건너뛴다. |
| Firebase FCM | 토큰 등록, 알림 수신 서비스 구현 | payload는 `type`, `referenceId`, `title`, `body` 기준이다. |
| 온디바이스 AI | `assets/ml` 모델 파일 사용 | 런타임에 `filesDir/models`로 복사된다. |

## 8. AI 모델 자산 기준

- 현재 런타임에서 직접 사용하는 파일은 아래 4개다.
  - `ft_model.bin`
  - `xgb_model_android.json`
  - `feature_map.json`
  - `label_map.json`
- 위 파일들은 `fe/app/src/main/assets/ml`에 있어야 한다.
- 앱 실행 중에는 `AiModelAssetManager`가 이 파일들을 `filesDir/models`로 복사한다.
- `input_matrix.npy`, `output_matrix.npy`, `ft_labels.json`은 legacy 성격이므로 삭제 전 실제 참조 여부를 다시 확인한다.

## 10. 인수인계 시 최종 확인 항목

- `local.properties` 예시값이 현재 팀 환경과 맞는지
- `google-services.json` 보관 위치와 최신 SHA가 맞는지
- 실기기 테스트용 IP 기반 URL이 바뀌었는지
- 릴리즈용 API URL과 서명 절차가 별도 채널에서 관리되는지
- 수동 회귀 체크 결과를 `운영 체크리스트` 문서 기준으로 남겼는지
