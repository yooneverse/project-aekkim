# FE 화면명세서 기반 전체 체크리스트

> 작성일: 2026-03-19  
> 2026-03-28 갱신: 실제 라우트와 화면 구현 상태 기준으로 전면 정리  
> 코드 기준: `FE/app/src/main/java/com/ssafy/e106/app/navigation`, `FE/app/src/main/java/com/ssafy/e106/feature`

## 1. 문서 목적

- 화면명세와 현재 FE 라우트/화면 구현 상태를 같은 기준으로 본다.
- 더 이상 placeholder 기준 문서와 실제 코드가 어긋나지 않도록 유지한다.
- QA 체크포인트를 현재 구현 기준으로 남기고, 미해결 항목은 후속 수정으로 분리한다.

## 2. 문서 운영 상태

### 현재 구현 기준

- 이 문서는 현재 라우트 연결과 화면 구현 상태를 기준으로 작성한다.
- `SCR-002b`, `SCR-005`, `SCR-006`, `SCR-008`은 더 이상 placeholder 화면이 아니다.
- 알림 딥링크 `aekkim://promotions/{id}`, `aekkim://cancel/{id}`도 실제 라우트로 연결된다.

### 권장 기준

- 실행 환경 복구와 운영 인수인계는 `2026-03-29_FE_운영_유지보수_인수인계_가이드.md`를 함께 본다.
- 장애 대응과 변경 후 스모크 테스트는 `2026-03-29_FE_운영_체크리스트_및_트러블슈팅.md`를 함께 본다.
- 디자인 및 컴포넌트 목표 상태는 `2026-03-28_FE_디자인시스템_구현기준_및_문서정합성.md`, `2026-03-12_FE_fdesign_css_definition__v2.md`, `2026-03-12_FE_mobile_component_spec_v1.md`를 함께 본다.
- 본 문서는 화면 구현/QA 기준 문서이며, 이상적인 디자인 목표를 대신 설명하지 않는다.

### 후속 수정 필요 항목

- 온보딩 로그인 정합성: Kakao 실연동 범위와 Naver 버튼 동작/문구
- 약관 상세 진입 정책: 외부 링크 vs 앱 내부 문서 기준 정리
- 분석 화면 초기 `0원` 노출
- 대시보드 `검토 필요 결제` 배너 문맥 강화
- 프로모션/추천/상세 계열 gradient 예외와 탭 라벨 통일

## 4. 공통 QA 준비

### 4-1. 실행 환경

- [ ] `FE/local.properties`의 `BASE_URL_DEV`를 실제 백엔드 주소로 맞춘다.
- [ ] `FE/app/google-services.json`이 존재한다.
- [ ] Android SDK Build Tools `35.0.0`이 설치되어 있다.
- [ ] 테스트 단말 또는 에뮬레이터에 Google 계정이 로그인되어 있다.

### 4-2. 권장 데이터 세트

- [ ] 신규 계정 1개: 온보딩, 분석, 구독 0건 확인용
- [ ] 구독 2건 이상 계정 1개: 대시보드, 상세, 인사이트, 추천 탭 검증용
- [ ] 미매핑 결제 1건 이상 계정 1개: `SCR-009` 검증용
- [ ] 체크인 미응답/부정 응답 계정 1개: `SCR-007`, `SCR-007-2`, `SCR-008` 검증용
- [ ] `CHECKIN`, `PROMO`, `CHURN_REVIEW` 알림이 있는 계정 1개: 알림함 검증용

## 6. 주요 화면 QA 체크리스트

### 6-1. `SCR-001` 온보딩 로그인

- [ ] Google 로그인 성공 시 약관 화면으로 이동한다.
- [ ] Google 로그인 실패 시 토스트가 보인다.
- [ ] Kakao 버튼은 현재 구현 정책대로 동작한다.
- [ ] Naver 버튼은 현재 `준비중 토스트`를 노출한다.

현재 갭

- 화면명세 관점의 소셜 로그인 목표와 실제 동작이 아직 완전히 일치하지 않는다.

### 6-2. `SCR-001-2` 약관 동의

- [ ] 필수 약관 2개 미동의 상태에서는 `동의하고 시작하기` 버튼이 비활성이다.
- [ ] 전체 동의와 개별 동의가 동기화된다.
- [ ] 알림 권한 다이얼로그에는 `지금은 건너뛰기`가 있다.
- [ ] UsageStats 권한 다이얼로그는 설정 이동만 제공하고 dismiss가 되지 않는다.
- [ ] UsageStats 권한 허용 후에만 분석 화면으로 진입한다.

현재 갭

- 약관 상세의 외부 링크 정책이 아니라 앱 내부 문서 화면 중심으로 운영된다.
- 알림 권한과 UsageStats 권한의 탈출 정책이 다르다.

### 6-3. `SCR-002` 절약 분석 중

- [ ] UsageStats 권한이 없으면 권한 안내 상태가 먼저 보인다.
- [ ] 권한 허용 후 분석 단계 3개가 순차적으로 진행된다.
- [ ] 분석 완료 시 `SCR-002b`로 이동한다.
- [ ] 수집 초기에도 금액 카드가 노출되는 현재 동작을 확인한다.

현재 갭

- 초기 수집 단계에서 `0원` 노출 가능성이 남아 있다.

### 6-4. `SCR-002b` 구독 확인

- [ ] 감지 구독 리스트가 카드 형태로 노출된다.
- [ ] 각 카드의 `구독 아님` 액션이 즉시 반영된다.
- [ ] 확인 필요 결제가 있으면 요약 카드와 `확인하기` CTA가 보인다.
- [ ] 하단 고정 `확인완료` CTA로 대시보드 진입이 가능하다.
- [ ] 감지 구독이 0건이어도 대시보드 진입이 가능하다.

현재 갭

- 제외 결과는 현재 분석 세션에만 반영된다.

### 6-5. `SCR-003` 대시보드

- [ ] 로딩/에러/Empty/기본 상태가 각각 분리되어 보인다.
- [ ] 알림 아이콘 탭 시 알림함으로 이동한다.
- [ ] `추가`와 `편집` 액션이 정상 동작한다.
- [ ] 체크인 배너는 `SCR-007`, 확인 필요 결제 배너는 `SCR-009`로 이동한다.
- [ ] 하단 탭 라벨이 대시보드에서는 `프로모션`으로 표시되는 현재 상태를 확인한다.

현재 갭

- 첫 카드가 총액 요약이라 행동 유도 우선순위가 약하다.
- 확인 필요 결제 배너는 count 중심이고 preview 문맥이 없다.

### 6-6. `SCR-004` 구독 상세

- [ ] 바텀시트로 열린다.
- [ ] 서비스명, 요금제, 금액, 다음 결제일이 보인다.
- [ ] 프로모션 상세와 해지 지원으로 이동할 수 있다.
- [ ] 딤 탭/스와이프로 닫힌다.

### 6-7. `SCR-005`, `SCR-006` 프로모션 추천

- [ ] `PromotionListRoute`, `PromotionDetailRoute`가 실제 라우트로 연결된다.
- [ ] 추천 목록에서 카테고리/추천유형 탭이 동작한다.
- [ ] 카드 hero 영역의 gradient, overlay, 블랙 savings badge가 현재 스타일로 노출된다.
- [ ] 상세 화면에서 가격 비교, 적용 절차, 외부 링크 CTA가 동작한다.
- [ ] 링크 없음/종료 상태에서는 CTA가 비활성화된다.

현재 갭

- 권장 디자인 시스템 대비 gradient/tint 예외가 강하다.
- 탭 라벨은 이 화면에서 `추천`을 사용해 대시보드와 용어가 다르다.

### 6-8. `SCR-007`, `SCR-007-2`, `SCR-008`

- [ ] `SCR-007`은 딥링크 `isFullScreen=true`일 때 뒤로가기가 막힌다.
- [ ] 부정 응답 누적 시 `SCR-007-2`로 이동한다.
- [ ] `SCR-007-2` Primary CTA는 `SCR-008`로 이동한다.
- [ ] `SCR-008`에서 해지 가이드 링크, 고객센터 전화, 문의 메일이 조건부로 노출된다.
- [ ] `나중에 할게요`로 이전 화면 복귀가 가능하다.

현재 갭

- 해지 지원은 구현됐지만 서비스별 메타데이터 품질에 따라 노출 편차가 있다.

### 6-9. `SCR-009` 수동 확인

- [ ] 카드에 가맹점명, 금액, 결제일, 추천 서비스명이 보인다.
- [ ] `서비스 선택` 탭 시 추가 플로우가 열린다.
- [ ] `구독 아님` 탭 시 현재 세션 로컬 상태에서 목록에서 빠진다.
- [ ] 목록을 모두 처리하면 Empty 상태가 보인다.

현재 갭

- `구독 아님` 처리는 영구 저장이 아니라 현재 세션 중심이다.

### 6-10. `SCR-010` 마이페이지

- [ ] 프로필, 이메일, provider 라벨이 보인다.
- [ ] 체크인/혜택/선택 정보 활용 토글이 저장된다.
- [ ] 약관/정책 상세로 이동한다.
- [ ] 로그아웃 다이얼로그와 회원 탈퇴 진입이 동작한다.

현재 갭

- 문의하기 채널은 아직 별도 화면으로 제공되지 않는다.

## 7. 알림/딥링크 검증 포인트

### 7-1. 알림함

- [ ] 목록 조회와 읽음 처리 API가 정상 동작한다.
- [ ] `CHECKIN`은 체크인 또는 관련 흐름으로 이동한다.
- [ ] `CHURN_REVIEW`는 구독 상세 관련 흐름으로 이동한다.
- [ ] `PROMO`는 실제 프로모션 상세로 이동한다.

### 7-2. 딥링크

```bash
adb shell am start -W -a android.intent.action.VIEW -d "aekkim://checkin/101?isFullScreen=true" com.ssafy.e106
adb shell am start -W -a android.intent.action.VIEW -d "aekkim://subscriptions/101" com.ssafy.e106
adb shell am start -W -a android.intent.action.VIEW -d "aekkim://promotions/501" com.ssafy.e106
adb shell am start -W -a android.intent.action.VIEW -d "aekkim://cancel/101" com.ssafy.e106
```

- [ ] `checkin` 딥링크가 체크인 풀스크린으로 열린다.
- [ ] `subscriptions` 딥링크가 구독 상세 바텀시트로 열린다.
- [ ] `promotions` 딥링크가 프로모션 상세로 열린다.
- [ ] `cancel` 딥링크가 해지 지원 화면으로 열린다.
