# FE 체크인 배치 연동 가이드

## 1. 문서 목적

- 프론트가 체크인 알림 배치 동작을 이해하고, 알림 화면/FCM 수신 흐름을 구현할 때 참고하기 위한 문서다.
- 아래 내용은 현재 구현 기준이다.

## 2. 체크인 배치 전체 흐름

### 2.1 04:00 체크인 대상 선정

- 매일 새벽 04:00에 CHECKIN 대상 선정 Batch가 실행된다.
- 선정 조건은 아래와 같다.
  - `nextBillingDate`가 오늘 기준 `D-3`
  - `subscriptions.low_usage_detected = true`
  - `subscriptions.low_usage_cycle_ym = 현재월`
  - `users.checkin_alert_enabled = true`
  - 해당 월에 이미 체크인 응답한 이력이 없음
  - 같은 CHECKIN 후보 알림이 이미 저장되어 있지 않음

### 2.2 09:00 체크인 알림 발송

- 매일 오전 09:00에 CHECKIN 발송 Batch가 실행된다.
- 04:00에 선정된 CHECKIN 후보 중 `sentAt = null`인 알림만 발송 대상으로 본다.
- 사용자 최신 FCM 토큰이 있는 경우에만 실제 FCM을 발송한다.
- 발송 성공 시 `notifications.sent_at`이 기록된다.

## 3. 프론트가 알아야 하는 동작 규칙

### 3.1 새벽 04:00에 후보가 저장돼도 알림함에는 바로 보이지 않는다

- 04:00 대상 선정 단계에서는 `notifications`에 CHECKIN 후보 row가 먼저 저장될 수 있다.
- 하지만 이 시점의 row는 `sentAt = null` 상태다.
- 알림 목록 조회 API는 `sentAt is not null`인 알림만 내려주도록 처리되어 있다.
- 따라서 프론트 알림함에는 **실제 발송된 CHECKIN 알림만** 보인다.

즉 정리하면:

- `04:00` : 후보 선정만 수행
- `09:00` : 실제 발송
- 알림함 노출 시점 : **09:00 발송 성공 후**

### 3.2 같은 체크인 알림이 중복 발송되지 않도록 방어하고 있다

- 체크인 발송 Batch는 Redis를 사용해 중복 발송을 방지한다.
- 기준은 `notificationId`가 아니라 **같은 의미의 CHECKIN 알림**이다.
- 현재 논리 기준은 아래 조합이다.
  - `userId`
  - `referenceId(subscriptionId)`
  - `body` 내 cycle 정보

따라서 같은 사용자에게 같은 구독, 같은 월에 대한 CHECKIN 알림 row가 중복 생성되더라도,
실제 FCM은 한 번만 발송되도록 방어한다.

### 3.3 아직 발송되지 않은 알림은 프론트에서 일반 알림처럼 보여주면 안 된다

- `sentAt = null`은 "후보 저장 상태"다.
- 이 값은 "사용자에게 이미 전달된 알림"이 아니다.
- 프론트는 알림함/읽지 않음 뱃지/알림 개수 표시를 모두 **실제 발송된 알림 기준**으로 생각하면 된다.

## 4. 프론트 연계 포인트

### 4.1 FCM 수신

- 오전 09:00 발송 Batch에서 실제 FCM이 발송된다.
- 프론트는 기존 FCM 수신 로직으로 CHECKIN 알림을 처리하면 된다.

### 4.2 알림 목록 조회

- 알림 목록 API에는 발송 완료된 CHECKIN 알림만 노출된다.
- 따라서 프론트는 별도 후보 상태를 고려하지 않고, 기존 알림 목록 화면에 그대로 표시하면 된다.

### 4.3 체크인 화면 진입

- 사용자가 CHECKIN 알림을 눌렀을 때는 `referenceId = subscriptionId`를 기준으로 해당 구독 화면/체크인 화면으로 이동시키면 된다.

## 5. 구현 범위 요약

이번 구현 범위는 아래 4개다.

1. CHECKIN 대상 선정 Batch 추가
2. CHECKIN 알림 발송 Batch 추가
3. Batch 실패 처리 로직 보강
4. Redis 기반 중복 실행/중복 발송 방지 로직 반영

## 6. 프론트 확인 포인트

- 오전 09:00 이전에는 CHECKIN 알림이 목록에 보이지 않는지
- 같은 월에 같은 구독 CHECKIN 알림이 2번 오지 않는지
- FCM 탭 시 해당 구독 상세/체크인 화면으로 정상 이동하는지
