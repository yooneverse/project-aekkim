# FE 디자인시스템 구현기준 및 문서정합성

> 작성일: 2026-03-28  
> 목적: `FE/app` 현재 구현을 기준으로 활성 문서를 다시 맞추고, 권장 기준 문서와 실제 구현을 분리해 관리한다.

## 1. 문서 역할

### 현재 구현 기준

- 이 문서는 2026-03-28 시점 `FE/app` 실제 코드 기준의 활성 베이스라인이다.
- 현재 화면과 공용 컴포넌트를 설명할 때는 이 문서를 먼저 본다.

### 권장 기준

- `2026-03-12_FE_fdesign_css_definition__v2.md`는 목표 디자인 토큰/스타일 규칙 문서다.
- `2026-03-12_FE_mobile_component_spec_v1.md`는 목표 컴포넌트 규격 문서다.
- 위 두 문서는 현재 구현 문서가 아니라, 정리해야 할 목표 상태 문서로 본다.

### 후속 수정 필요 항목

- Pretendard 적용 후 전체 QA
- `TextSecondary(#8E8E93)` 재정의와 보조 텍스트 대비 상향
- 하단 탭바 active indicator 추가 및 탭 라벨 통일
- 프로모션/추천/상세 계열 gradient/tint 예외 정리
- 권한 다이얼로그 예외 스타일과 dismiss 정책 정리
- 화면 구현 문서와 실제 라우트 상태 지속 동기화

## 3. 현재 구현 기준

### 3-1. 테마와 토큰

- Light mode only 운영: `Theme.kt`
- white base: `Background`, `Surface` 모두 `#FFFFFF`
- primary: `#F5A623`, pressed: `#E99A14`, success: `#22C55E`
- 보조 텍스트: `TextSecondary = #8E8E93`, `onSurfaceVariant`에 직접 매핑
- 폰트: `Type.kt`가 `PretendardFontFamily`로 전체 사용 토큰과 기본 토큰을 연결한다.
- 타이포 스케일은 권장 문서와 완전히 같지 않다.
  - 예: `displayLarge`는 현재 `36sp / 50sp / FontWeight.Black`

### 3-2. 공용 컴포넌트

| 영역 | 현재 구현 기준 |
| --- | --- |
| 버튼 | `AppButton`은 `Primary`, `Secondary`, `Ghost` 3종만 제공한다. Social 로그인은 `feature/auth/ui/component/SocialLoginButton.kt`로 분리돼 있다. |
| 카드 | `AppCard`는 기본 20dp radius, 1dp outline border를 가진다. `CardVariant`는 `Default`, `Elevated`만 제공한다. |
| 바텀시트 | `AppBottomSheet`는 white surface, drag handle, scrollable 옵션을 제공하고 바깥 탭/뒤로가기 dismiss가 가능하다. |
| 탭바 | `AppBottomTabBar`는 높이 72dp, top divider, selected tint만 제공한다. active indicator는 없다. |
| spacing/shape | `Dimens.kt`와 `Shape.kt`는 4/8pt grid와 8/12/16/20 radius 체계를 사용한다. |

### 3-3. 화면별 예외 스타일

- 약관 화면
  - 진행 패널과 약관 카드에 gradient/tint 배경이 들어간다.
  - 권한 다이얼로그는 gradient 헤더를 사용한다.
  - 알림 권한 다이얼로그만 `지금은 건너뛰기` 보조 액션이 있다.
  - UsageStats 다이얼로그는 보조 액션 없이 설정 이동만 허용한다.
  - 권한 다이얼로그는 바깥 탭/백버튼 dismiss가 불가능하다.
- 분석 화면
  - `SavingsAmountCard`가 항상 렌더링되어 초기 수집 중에도 `0원`이 노출될 수 있다.
- 대시보드
  - 첫 카드가 총액 요약 카드다.
  - `검토가 필요한 결제 내역 N건` 배너는 count 중심이고 preview 문맥은 없다.
  - 하단 탭 라벨은 `프로모션`을 사용한다.
- 프로모션/추천/상세 계열
  - 추천 카드 hero와 번들 hero에 브랜드 gradient가 적용된다.
  - 이미지 위 overlay와 블랙 savings badge가 함께 쓰인다.
  - 프로모션 상세는 화이트 베이스가 중심이지만 savings badge와 가격 강조가 primary 중심으로 강하다.
- 체크인/후속
  - 체크인 응답 버튼은 `AppButton`이 아니라 `OutlinedButton + SurfaceMuted` fill 조합이다.
  - 후속 화면은 `AppButton` Primary/Secondary 조합을 사용한다.

## 4. 권장 기준

- Pretendard는 실제 리소스로 적용 완료됐고, 이후 typography 조정은 현재 token 수치(fontSize/lineHeight)를 유지한 채 화면 QA 중심으로 진행한다.
- `TextSecondary`를 단일 전역값으로 두지 않고 최소 2단계로 분리해 설명문과 placeholder 대비를 올린다.
- 하단 탭바에 active indicator를 추가하고 `프로모션`/`추천` 라벨을 한 방향으로 통일한다.
- 프로모션/추천/상세의 gradient는 hero 예외로 축소하고, 본문 카드와 조건 카드에서는 white base를 유지한다.
- 권한 다이얼로그는 gradient/tint 예외를 줄이고, dismiss 정책과 탈출 경로를 권한별로 다시 정리한다.
- 화면 구현 문서는 route 연결 여부와 실제 UI 상태를 기준으로 유지한다.
