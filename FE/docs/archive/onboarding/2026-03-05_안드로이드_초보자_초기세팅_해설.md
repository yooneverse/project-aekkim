# 안드로이드 초보자를 위한 FE 초기세팅 해설

이 문서는 `FE/docs/FE_초기세팅_가이드.md`를 처음 안드로이드 앱을 만드는 사람 관점으로 다시 풀어쓴 안내서입니다.

## 0) 먼저 큰 그림부터 이해하기

이 프로젝트는 다음 기술을 기준으로 시작합니다.

- Android 네이티브: Kotlin + Jetpack Compose
- 구조: MVVM + 단방향 상태 흐름(State/Intent)
- 핵심 라이브러리: Hilt(DI), Retrofit/OkHttp(네트워크), DataStore(토큰 저장), Firebase FCM(푸시)

즉, "화면만 먼저 대충"이 아니라, 처음부터 확장 가능한 기본 뼈대를 만드는 것이 목표입니다.

---

## 1) 설치 순서 (처음 하는 사람용)

### 1-1. 필수 도구 설치

1. Android Studio 설치
2. Gradle JDK를 17로 설정(Android Studio Embedded JDK 권장)
3. SDK Manager에서 Android SDK 설치 (프로젝트 기준 compile/target 35, min 26)
4. 에뮬레이터(AVD) 1개 생성

팁:
- SDK가 없으면 Gradle sync 단계에서 바로 실패합니다.
- 에뮬레이터를 먼저 만들어 두면 API 연동 확인이 빨라집니다.
- Kotlin 버전과 KSP 버전 앞자리가 다르면 Hilt 코드 생성이 실패할 수 있습니다.

### 1-2. 프로젝트 생성

Android Studio에서 새 프로젝트를 다음 값으로 만듭니다.

- Template: Empty Activity
- Package: `com.ssafy.e106`
- Language: Kotlin
- Minimum SDK: API 26
- Build config language: Kotlin DSL (`.kts`)

### 1-3. 프로젝트 생성 직후 바로 할 일

아래는 "코드 작성 전"에 끝내야 하는 항목입니다.

1. `.gitignore`에 `local.properties`, `*.jks` 포함 확인
2. `local.properties`에 아래 값 추가

```properties
BASE_URL_DEV=http://[dev서버IP]:8081
BASE_URL_PROD=http://[prod서버IP]:8082
```

3. `gradle/libs.versions.toml` 작성
4. 루트/앱 `build.gradle.kts` 의존성 반영
5. `AndroidManifest.xml` 권한/컴포넌트 선언
6. `App.kt`, `MainActivity.kt` 생성
7. `google-services.json` 추가(FCM)

## 2) "왜 이 순서인가?" (초보자가 자주 막히는 지점)

### 이유 1: 빌드 기반이 먼저다

의존성(`libs.versions.toml`, Gradle) 설정 없이 화면부터 만들면,
중간에 Hilt/Navigation/Serialization 설정 충돌로 다시 뜯어고치게 됩니다.

### 이유 2: 앱 시작점을 먼저 고정한다

- `App.kt`: 앱 전역 초기화(Hilt, NotificationChannel, 토큰 저장소 초기화)
- `MainActivity.kt`: Compose 루트 + NavHost 시작

시작점을 먼저 만들면 이후 feature를 붙일 때 경로가 명확합니다.

### 이유 3: 인증/푸시는 초기 구조와 강하게 연결된다

- 소셜 로그인 provider SDK별 리다이렉트/스킴 요구사항 확인 필요 `[추가 필요]`
- FCM: `google-services.json` + 메시지 서비스 등록 필요

이 둘은 나중에 붙이면 설정 누락을 찾기 어렵습니다.

## 3) 처음 작성할 때의 폴더 구조

아래처럼 "레이어 분리"가 핵심입니다.

```text
app/src/main/java/com/ssafy/e106/
├── app/          # 앱 시작점, 네비게이션
├── core/         # 공통 UI, 모델, 유틸, Result
├── di/           # Hilt 모듈
├── data/         # API, DTO, Repository, Local(DataStore), Interceptor
└── feature/      # 기능별 화면 단위(auth, dashboard, ...)
```

초보자 관점 해석:
- `feature/`는 "화면 기능별 작업 공간"
- `data/`는 "서버/로컬 데이터 처리 공간"
- `core/`는 "여러 화면에서 재사용하는 공용 공간"

## 4) 기능 하나를 작성하는 고정 패턴

새 기능을 만들 때는 항상 같은 4파일로 시작합니다.

1. `XxxScreen.kt` (UI)
2. `XxxViewModel.kt` (상태/로직)
3. `XxxState.kt` (화면 상태 모델)
4. `XxxIntent.kt` (사용자 액션)

흐름은 아래 하나로 통일합니다.

```text
Screen -> ViewModel(onIntent) -> Repository -> API/Local
       <-      State/Event      <-
```

이 패턴을 지키면 "화면에서 API 직접 호출" 같은 유지보수 난이도 높은 코드를 피할 수 있습니다.

## 5) 실제 개발 진행 순서 (추천)

`FE_초기세팅_가이드.md`의 로드맵을 초보자용으로 줄이면 아래 순서가 가장 안전합니다.

1. 프로젝트 뼈대 완성
   - Gradle/의존성
   - App/MainActivity/NavHost
   - core/di/data 기본 파일
2. 인증 흐름 먼저 완성
   - auth feature
- 소셜 로그인 인증값 교환 -> JWT 저장(DataStore)
3. 홈 핵심 화면 추가
   - dashboard/analysis
4. 부가 기능 추가
   - promotion/checkin/cancel/mapping
5. 마이페이지 + 안정화
   - settings
   - 에러 처리 통일, 테스트, lint/typecheck

핵심은 "한 번에 모든 화면"이 아니라 "기능 1개를 끝까지"입니다.

## 6) 실행/연동 체크포인트

### 앱 기동 직후 확인

- 앱이 크래시 없이 시작되는가
- NavHost 시작 화면이 정상 표시되는가
- Hilt 주입 에러가 없는가

### API 연동 시작 시 확인

- 에뮬레이터에서 로컬 서버 접근 시 `10.0.2.2` 사용 여부
- `BASE_URL`이 buildType(debug/release)에 맞게 주입되는지
- 401 발생 시 토큰 삭제 후 인증 화면으로 이동하는지

### FCM/Auth 확인

- `google-services.json` 위치가 app 모듈 기준인지
- Manifest에 앱 딥링크(`aekkim://`)와 FCM 서비스 선언이 정확한지
- Android 13+ 알림 권한 처리 흐름이 있는지

추가로 아래 5개를 꼭 확인하세요.

1. provider 인증값(`authorizationCode`/`providerToken`)과 앱 JWT(`accessToken`/`refreshToken`)를 혼용하지 않았는지
2. 소셜 로그인 교환 endpoint/요청 필드가 `[협의 필요: FE/BE]`로 분리 관리되는지
3. provider SDK 세부 방식(Credential Manager 포함 여부)을 기준 문서 없이 임의 확정하지 않았는지
4. Firebase `google-services.json`의 `package_name`과 `applicationId`가 일치하는지
5. API 33+ 기기에서 `POST_NOTIFICATIONS` 런타임 권한 요청 코드가 있는지

## 7) 초보자 실수 방지 규칙 (중요)

아래 5가지는 반드시 지키면 됩니다.

1. Screen에서 Repository/Retrofit 직접 호출 금지
2. ViewModel 외부로 `MutableStateFlow` 노출 금지
3. 화면 액션은 `onIntent(...)` 하나로 통일
4. 로딩/에러/빈 상태 3종 처리 필수
5. 새 기능은 새 `feature/{name}` 폴더로 분리

---

## 8) 오늘 바로 시작할 최소 할 일

1. Android Studio로 프로젝트 생성
2. `libs.versions.toml` + Gradle 반영
3. `App.kt`, `MainActivity.kt`, `AppNavHost.kt` 생성
4. `TokenDataStore`, `AuthInterceptor`, `NetworkModule` 작성
5. `feature/auth` 4파일(State/Intent/ViewModel/Screen) 생성

여기까지 되면 이후 기능 추가는 같은 패턴 반복으로 진행할 수 있습니다.
