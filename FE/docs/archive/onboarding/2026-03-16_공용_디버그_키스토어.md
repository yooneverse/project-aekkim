# 공용 디버그 키스토어

안드로이드 디버그 빌드는 공용 키스토어 `FE/signing/debug.keystore`를 사용한다.

## Firebase 등록값

Firebase 앱 `com.ssafy.e106`에 아래 지문을 등록한다.

- SHA-1: `05:31:54:21:5C:54:51:6F:0C:05:1C:BB:21:6F:FF:1C:A6:92:01:D7`
- SHA-256: `63:6C:D0:55:71:15:DF:94:1D:E1:29:C9:6C:BC:BC:CB:14:EE:D6:0A:A5:92:40:22:17:DE:08:B1:CF:91:9F:31`

## 문서에 무엇을 적는가

- 문서에는 Firebase 등록용 `SHA-1`, `SHA-256` 지문만 기록한다.
- 개인키 원문이나 별도 비밀값은 문서에 적지 않는다.
- 이번 설정은 디버그 전용 공용 키스토어이므로 팀 내부 공유 목적의 SHA 기록은 허용한다.

## 팀 작업 규칙

- 팀원은 PC마다 별도 디버그 SHA를 Firebase에 등록할 필요가 없다.
- 단, 모두가 저장소의 공용 키스토어로 디버그 빌드해야 한다.
- 기존에 다른 디버그 서명으로 설치한 앱이 있으면 기기에서 앱을 삭제한 뒤 다시 설치한다.
- Firebase Authentication 또는 Google Sign-In 설정이 바뀌면 `google-services.json`을 다시 내려받아 `FE/app/google-services.json`에 반영한다.
