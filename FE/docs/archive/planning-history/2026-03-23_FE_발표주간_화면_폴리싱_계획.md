# FE 발표주간 화면 폴리싱 계획

> 작성일: 2026-03-23  
> 범위: `[FE] (Subscription) 구독확인-대시보드-해지지원 화면 정비`, `[FE] (UX) 최종 발표용 화면 폴리싱`

## 1. 문서 목적

- 발표용 시연 화면을 기능 중심 구현 상태에서 발표용 완성도 수준으로 끌어올린다.
- 구독확인 → 대시보드 → 구독상세 → 해지지원 핵심 흐름의 시각적/문구적 일관성을 맞춘다.
- 공통 디자인 규칙을 기준으로 화면별 예외 스타일과 어색한 상태 화면을 정리한다.

## 2. 폴리싱 목표

### 2-1. Subscription 축

- `SCR-002b` 구독 확인 화면 마감
- OTT 6종 노출/정렬/브랜딩 정합성 통일
- 구독 상세 CTA 슬롯 확장
- `SCR-008` 해지지원 화면과 진입 흐름 정리

### 2-2. UX 축

- 문구/라벨/띄어쓰기/톤앤매너 일괄 점검
- 여백/정렬/버튼 상태/UI 일관성 수정
- `Loading/Empty/Error` 상태 화면 통일
- 최종 디자인 QA 반영

## 3. 폴리싱 원칙

- 배경은 `#FFFFFF` 기준으로 유지한다.
- 강조는 Primary 오렌지와 상태용 초록만 사용한다.
- 카드, 버튼, 바텀시트, 상태 화면은 공통 컴포넌트 규칙을 우선 적용한다.
- 색으로 덮는 방식보다 간격, 정렬, 보더, 타이포그래피로 완성도를 높인다.
- 발표 시연 화면은 "한 번 보고 이해되는 구조"를 우선한다.

## 4. 우선순위

### 1순위. 핵심 시연 플로우 화면

- `SCR-002b` 구독 확인
- `SCR-003` 대시보드
- `SCR-004` 구독 상세
- `SCR-008` 해지지원

### 2순위. 연결 플로우 화면

- `SCR-009` 수동확인
- `SCR-007`, `SCR-007-2` 체크인/후속 권유
- 알림함
- 프로모션 목록/상세

### 3순위. 공통 화면

- 온보딩/약관
- 마이페이지
- 공통 상태 화면

## 5. 세부 작업 계획

### 5-1. 문구/라벨/톤앤매너 정리

#### 목표

- 발표 청중이 보기에 어색한 카피, 중복 표현, 버튼 라벨 불일치를 없앤다.

#### 점검 항목

- `구독 아님`, `OTT 구독 맞아요`, `확인완료` 문구 정렬
- `혜택`, `프로모션`, `추천` 용어 통일
- `해지지원`, `해지 방법 보기`, `나중에 할게요` 라벨 톤 정리
- 에러 토스트 문구 길이와 말투 통일
- 띄어쓰기와 조사 어색함 정리
- Empty 상태 설명 문구를 기능별로 짧고 명확하게 정리

#### 화면별 우선 점검

| 화면 | 중점 |
| --- | --- |
| `SCR-002b` | 구독 검토 문구, CTA 문구, Empty 안내 |
| `SCR-003` | 배너 카피, 편집/추가/새로고침 라벨 |
| `SCR-004` | 상세 CTA 라벨, 추천 문구 |
| `SCR-008` | 해지지원 설명, 연락 수단 안내 |
| `SCR-009` | 수동확인 버튼 라벨 |
| 알림함 | 타입별 제목/본문 요약 |

#### 완료 기준

- 같은 의미의 버튼이 화면마다 다른 용어를 쓰지 않는다.
- 토스트, 다이얼로그, 배너 카피가 한 톤으로 읽힌다.

### 5-2. 여백/정렬/UI 일관성 수정

#### 목표

- 화면 간 카드, 버튼, 리스트, 바텀시트의 밀도와 정렬 기준을 맞춘다.

#### 공통 기준

- 화면 좌우 패딩: 기본 `20px`
- 카드 내부 패딩: 기본 `16px`
- 섹션 간 간격: 기본 `24px`
- 버튼 높이/radius: 공통 컴포넌트 기준 사용
- 리스트 trailing 정렬: 오른쪽 기준 통일

#### 점검 항목

- 카드 상하 여백 불균형
- CTA 버튼 폭/높이 불일치
- 서비스 로고와 텍스트 baseline 불일치
- 바텀시트 drag handle, title, body 간격 불일치
- Empty/Error 화면에서 콘텐츠 수직 정렬이 흔들리는 문제
- 편집 모드와 기본 모드의 리스트 높이 차이

#### 화면별 우선 점검

| 화면 | 중점 |
| --- | --- |
| `SCR-002b` | 리스트 스크롤 영역, 하단 고정 CTA |
| `SCR-003` | 배너, 카드, 섹션 간격, 탭/버튼 배치 |
| `SCR-004` | 서비스 정보와 CTA 영역 분리 |
| `SCR-008` | 연락 수단 섹션 정렬, 조건부 노출 시 레이아웃 안정성 |
| `SCR-009` | 카드 액션 정렬, 추가 플로우 진입부 |

#### 완료 기준

- 카드/버튼/리스트가 화면별로 제각각 보이지 않는다.
- 조건부 UI가 빠져도 레이아웃이 무너지지 않는다.

### 5-3. OTT 6종 노출/정렬/브랜딩 정합성

#### 목표

- 서비스 노출 화면 전반에서 로고, 명칭, fallback 정책을 통일한다.

#### 점검 항목

- `Netflix`, `Tving`, `Wavve`, `Coupang Play`, `Disney+`, `Watcha` 6종 명칭 통일
- `logoUrl` 존재 시 동일한 렌더링 규칙 사용
- `logoUrl` 미존재 시 첫 글자 fallback 정책 통일
- 서비스 카드, 수동확인, 구독확인, 해지지원에서 같은 브랜드 규칙 사용
- 서비스 정렬 순서와 노출 우선순위 통일

#### 완료 기준

- 동일 서비스가 화면마다 다른 이름/배지/정렬로 보이지 않는다.
- 브랜드 로고가 있는 곳과 없는 곳의 시각 규칙이 일관된다.

### 5-4. 구독 상세 CTA 슬롯 확장

#### 목표

- 구독상세가 단순 정보 바텀시트가 아니라 다음 행동으로 이어지는 화면이 되게 한다.

#### 포함 항목

- `해지 방법 보기`
- 고객센터 전화
- 문의 이메일
- 추천 프로모션 진입

#### 점검 항목

- CTA 우선순위가 자연스러운지
- 값이 없는 CTA는 숨김 또는 대체 문구로 처리되는지
- 버튼 스타일이 Primary/Secondary/Ghost 규칙과 맞는지
- 바텀시트 내 CTA가 3개 이상일 때 계층이 무너지지 않는지

#### 완료 기준

- 구독상세에서 다음 행동이 분명하게 보인다.
- 해지지원으로 이어지는 흐름이 설명 없이도 이해된다.

### 5-5. `SCR-008` 해지지원 화면 폴리싱

#### 목표

- 발표 시연에서 "서비스를 끝까지 관리할 수 있다"는 인상을 주는 화면으로 정리한다.

#### 점검 항목

- 서비스명과 브랜드 표시
- 해지 가이드 링크 노출 방식
- 전화/이메일 CTA 계층
- `나중에 할게요` 보조 행동 위치
- 링크/전화번호 부재 시 대체 문구
- 복귀 후 컨텍스트 유지

#### 완료 기준

- 정보가 적어도 허전하지 않고, 많아도 복잡하지 않다.
- 링크/전화/메일 유무별 케이스가 모두 자연스럽다.

### 5-6. Loading/Empty/Error 상태 화면 통일

#### 목표

- 상태 화면이 기능마다 따로 놀지 않도록 공통 패턴을 맞춘다.

#### 공통 원칙

- `Loading`: skeleton 또는 진행 메시지 중심
- `Empty`: 설명 + 보조 CTA
- `Error`: 문제 설명 + 재시도 CTA
- 배경은 항상 흰색 유지
- 아이콘/일러스트/카피 밀도를 통일

#### 우선 적용 화면

- 대시보드
- 구독확인
- 수동확인
- 알림함
- 프로모션 목록/상세
- 해지지원

#### 완료 기준

- 상태만 봐도 사용자가 다음 행동을 이해할 수 있다.
- 기능별 상태 화면이 다른 앱처럼 보이지 않는다.

## 6. 화면별 실행 순서

1. `SCR-002b` 구독확인 구조와 카피 확정
2. `SCR-003` 대시보드 카드/배너/상태 화면 정리
3. `SCR-004` 구독상세 CTA 구조 정리
4. `SCR-008` 해지지원 정보 구조 정리
5. `SCR-009` 수동확인 버튼/카피/상태 정리
6. `SCR-007`, `SCR-007-2` 체크인/후속 권유 문구와 CTA 정리
7. 알림함/프로모션 상태 화면 정리
8. 온보딩/마이페이지 마감 polish

## 7. 디자인 QA 체크리스트

### 7-1. 공통

- [ ] 배경이 흰색 기준으로 유지된다.
- [ ] Primary 오렌지 사용이 CTA/강조 수치에 제한된다.
- [ ] 버튼 높이와 radius가 공통 규격과 맞다.
- [ ] 카드 padding과 섹션 간격이 일관된다.
- [ ] 상태가 색상만으로 구분되지 않는다.

### 7-2. Subscription 플로우

- [ ] `SCR-002b`의 하단 CTA가 안정적으로 고정된다.
- [ ] 대시보드 배너와 카드 계층이 명확하다.
- [ ] 구독상세 CTA 우선순위가 자연스럽다.
- [ ] 해지지원 화면이 값 유무에 따라 자연스럽게 변형된다.
- [ ] OTT 6종 브랜딩이 화면별로 다르지 않다.

### 7-3. UX 플로우

- [ ] 문구/라벨/띄어쓰기가 일관된다.
- [ ] `Loading/Empty/Error` 상태 패턴이 통일된다.
- [ ] 토스트와 다이얼로그 톤이 과하게 다르지 않다.
- [ ] 버튼 상태와 비활성 표시가 명확하다.

## 8. 완료 판정 기준

- 발표 핵심 플로우 화면이 기능적으로뿐 아니라 시각적으로도 하나의 제품처럼 보인다.
- `SCR-002b`, `SCR-003`, `SCR-004`, `SCR-008` 흐름에서 거슬리는 문구/정렬/브랜딩 불일치가 사라진다.
- `Loading/Empty/Error` 상태 화면이 공통 규칙으로 정리된다.
- 최종 디자인 QA 피드백을 반영할 수 있는 수정 우선순위가 문서에 정리된다.
