# FE 단독 실행 태스크 백로그

> 작성일: 2026-03-15  
> 목적: 현재 시점에서 BE 신규 구현을 기다리지 않고 FE가 자체적으로 바로 수행할 수 있는 작업만 우선순위별로 정리한다.  
> 선정 기준: `현재 API가 이미 존재하거나`, `mock/fixture만으로 화면 완성이 가능하거나`, `FE 내부 계약/네비게이션/상태 정리만으로 끝낼 수 있는 작업`만 포함한다.

## 1. 현재 판단 기준

- 포함
  - FE 코드만 수정하면 되는 작업
  - 이미 존재하는 인증/사용자 API를 붙이는 작업
  - 구독/알림/체크인 영역이라도 mock UI로 먼저 완성 가능한 작업
- 제외
  - BE가 새 endpoint를 구현해야 시작 가능한 작업
  - provider 필드처럼 BE 응답 변경이 선행되어야 하는 작업
  - FCM 토큰 등록처럼 API 계약이 아직 비어 있는 작업

## 6. 지금 당장 착수하지 않는 작업

- Kakao 로그인 실연동
  - FE 작업 비중이 크지만 SDK 설정/앱 키/운영 환경 확인이 필요해 즉시 착수 항목에서 제외했다.
- provider 배지 완성
  - BE 응답에 provider 필드가 없어 FE 단독 완료가 불가능하다.
- FCM 토큰 등록
  - BE API 계약이 아직 비어 있다.
- 구독/프로모션 실API 연결
  - 현재 BE 구현체가 없다.
