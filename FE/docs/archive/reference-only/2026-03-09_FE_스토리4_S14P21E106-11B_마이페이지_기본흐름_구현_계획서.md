### 필수 상태

- 프로필 조회 실패 시 기본값 fallback 허용
- 로그아웃/탈퇴 처리 중 버튼 비활성
- 알림 토글은 UI 슬롯과 초기값 정리까지만 Story4 범위
- 문의하기는 mailto/외부 URL 중 하나로 연결하되 실제 채널은 `[확인 필요]`

### 디자인 / 컴포넌트 원칙

- White Base, 보더 중심 계층, 카드/리스트 행 공통 컴포넌트 우선
- destructive 색상은 확정 전까지 일반 danger fill을 새로 만들지 않는다
- 로그아웃/탈퇴 버튼은 공통 버튼 variant 조합으로 처리한다

## 4. ViewModel / Repository / API 범위

### ViewModel 책임

`MyPageViewModel`

- 프로필 조회 및 state 반영
- 앱 버전 문자열 반영
- 로그아웃/탈퇴 다이얼로그 상태 제어
- Story5 연결용 토글 action 수용 포인트 제공

### Repository 책임

- `UserRepository`: `getMe`, `deleteMe`
- `AuthRepository` 또는 전용 세션 repository: `logout`
- `TokenRepository`: logout/delete 후 clear

### API 목록

| API | 상태 | 비고 |
| --- | --- | --- |
| `GET /api/v1/users/me` | 확정 | 프로필 조회 |
| `DELETE /api/v1/users/me` | 확정 | 회원 탈퇴 |
| `POST /api/v1/auth/logout` | 확정 | 로그아웃 |
| `PATCH /api/v1/users/me/notification-settings` | 확정 | 실제 저장은 Story5 |

### 데이터 우선순위 규칙

- `displayName`, `email`, `profileImageUrl`의 정본은 `/users/me` 응답
- 로그인 직후 user 정보는 fallback 또는 초기 렌더링 보조값으로만 사용
- `checkinAlertEnabled`, `promoAlertEnabled`도 장기적으로는 `/users/me` 또는 알림 설정 API 응답을 정본으로 본다
