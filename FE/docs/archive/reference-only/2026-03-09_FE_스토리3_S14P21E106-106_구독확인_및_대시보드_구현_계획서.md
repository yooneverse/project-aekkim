### 필수 상태

- `SCR-002b` empty 상태에서도 `확인완료` CTA 유지
- 대시보드 로딩/에러/빈/재시도/새로고침/편집 모드
- 체크인 배너는 Story5 실데이터 연결 전까지 슬롯 상태로 유지
- 확인 필요 결제 배너는 count 기반 표시
- `selectedSubscriptionId` 기준 상세 바텀시트 오픈
- `manualAddStep` 기준 `SCR-003-2` -> `SCR-003-3` 전환

### CTA / 서비스 범위 충돌 메모

- 대시보드 CTA는 화면명세 내 `추가/편집`과 `구독정보 불러오기/구독 편집하기`가 충돌하므로 `[확인 필요: 기획]`로 관리한다.
- 수동 추가 서비스 목록은 요구사항상 OTT 5개가 기본이나 화면명세에는 6종(왓챠 포함) 시안이 남아 있어 `[확인 필요: 기획]`로 관리한다.

## 4. ViewModel / Repository / API 범위

### ViewModel 책임

`AnalysisViewModel` 확장 범위

- `SCR-002b` 후보 목록, 제거, 확정 action 관리

`DashboardViewModel`

- 구독 목록/월 총액 조회
- 편집 모드 전환
- 추가/상세/수동 확인 바텀시트 상태 관리
- 대시보드 재시도/새로고침 처리

### Repository 책임

- `SubscriptionRepository`: 목록/상세/생성/수정/삭제
- `ServiceRepository`: OTT 서비스 목록/요금제 조회
- pending review 상세 데이터는 별도 API 확정 전 `[확인 필요: FE/BE]`

### API 목록

| API | 상태 | 비고 |
| --- | --- | --- |
| `GET /api/v1/subscriptions` | 확정 | 대시보드 목록 + 월 총액 |
| `GET /api/v1/subscriptions/{subscriptionId}` | 확정 | 상세 바텀시트 |
| `POST /api/v1/subscriptions` | 확정 | 수동 추가 |
| `PATCH /api/v1/subscriptions/{subscriptionId}` | 확정 | 요금제/결제일 수정 |
| `DELETE /api/v1/subscriptions/{subscriptionId}` | 확정 | 제거 |
| `GET /api/v1/services?category=OTT` | 확정 | 서비스 선택 |
| `GET /api/v1/services/{serviceId}/plans` | 확정 | 요금제 조회 |
| 구독 후보 확정 저장 API | `[확인 필요: FE/BE]` | `SCR-002b` 저장 계약 미정 |
| pending review 상세 조회 API | `[확인 필요: FE/BE]` | `SCR-009` 실데이터 |
