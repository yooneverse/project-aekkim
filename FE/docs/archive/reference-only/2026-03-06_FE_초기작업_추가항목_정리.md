# FE 초기작업 추가항목 정리 (UsageStats 연동 기준)

> 작성일: 2026-03-06  
> 대상: Android FE 팀 (초기작업 이후 1~2주차)  
> 목적: 현재 초기세팅 상태에서 "실서비스 가능한 UsageStats 기능"까지 가기 위해 필요한 추가 작업을 우선순위로 정리

## 0. 문서 사용법

- 이 문서는 기존 초기세팅 문서를 대체하지 않고 "추가 백로그"를 정의한다.
- 우선순위는 `P0(즉시)`, `P1(스프린트 내)`, `P2(운영 전)`로 나눈다.
- 각 항목은 완료 기준(DoD)을 포함한다.

---

## 1. 현재 상태 요약 (레포 기준)

확인된 상태:

- `PACKAGE_USAGE_STATS` 권한 선언은 이미 반영됨 (`FE/app/src/main/AndroidManifest.xml`)
- FE 패키지 구조(`app/core/data/di/feature/...`)는 생성됐지만 다수는 스캐폴딩 단계
- `FE/docs/FE_초기세팅_가이드.md`에 로드맵/체크리스트는 있으나, UsageStats 실구현 단계 문서가 부족
- 초기작업 이슈 문서 기준으로 토큰/FCM/딥링크/릴리즈 보안 등 미완료 과제가 존재

결론:

- "설정 완료" 상태가 아니라 "구현 시작 가능" 상태다.
- UsageStats를 프론트에 실제 표시하려면 권한 플로우 + 데이터 수집 + API 계약 + 상태 UI + 테스트가 추가로 필요하다.

---

## 2. 먼저 고정해야 하는 팀 합의 (필수)

아래 5가지는 코딩 전에 먼저 고정해야 재작업을 줄일 수 있다.

1. **집계 기준**: 최근 7일/30일의 "로컬 날짜 버킷" 기준
2. **빈 데이터 정책**: `permission_denied`, `no_history`, `oem_restricted`를 구분
3. **표시 단위**: 내부 저장 ms, UI 표시는 시/분 포맷
4. **데이터 위치**: 앱 로컬 즉시 표시 + 서버 업로드 병행(권장)
5. **대상 앱 목록**: packageName allowlist를 어디서 관리할지(앱 상수 vs 서버 설정)

## 5. 프론트 상태 설계 추가안

UsageStats 기능은 아래 상태가 필수다.

1. `Loading`
2. `PermissionRequired`
3. `EmptyNoHistory`
4. `EmptyOemRestricted`
5. `Content`
6. `Error`

최소 상태 모델 예시:

```kotlin
data class UsageStatsUiState(
    val isLoading: Boolean = false,
    val permissionGranted: Boolean = false,
    val reason: String? = null,
    val items: List<AppUsageItemUi> = emptyList(),
    val errorMessage: String? = null,
    val updatedAtEpochMs: Long? = null
)
```

## 6. API 계약 초안 (FE 관점)

요청(예):

```json
{
  "timezone": "Asia/Seoul",
  "queriedAtEpochMs": 1767686400000,
  "items": [
    {
      "packageName": "com.netflix.mediaclient",
      "usage7dMs": 5400000,
      "usage30dMs": 18200000,
      "lastUsedEpochMs": 1767643200000,
      "permissionGranted": true,
      "reason": null
    }
  ]
}
```

응답(예):

```json
{
  "success": true,
  "data": {
    "permissionGranted": true,
    "reason": null,
    "updatedAtEpochMs": 1767686401234,
    "items": [
      {
        "packageName": "com.netflix.mediaclient",
        "usage7dMs": 5400000,
        "usage30dMs": 18200000,
        "lastUsedEpochMs": 1767643200000
      }
    ]
  },
  "message": null
}
```

FE 처리 규칙:

- `permissionGranted=false` -> 권한 안내 UI
- `permissionGranted=true && items=[] && reason=no_history` -> 데이터 없음 UI
- `permissionGranted=true && items=[] && reason=oem_restricted` -> 기기 제한 안내 UI

## 8. Definition of Done (이 문서 기준)

아래가 모두 충족되면 "초기작업 추가 반영 완료"로 본다.

- [ ] UsageStats 권한 플로우가 실제 기기에서 end-to-end 동작
- [ ] 최근 7일/30일/마지막 사용일이 최소 1개 앱에서 정상 표시
- [ ] 거절/철회/데이터없음/OEM제한을 UI에서 구분 노출
- [ ] FE-BE API 계약 문서와 실제 DTO가 일치
- [ ] 핵심 테스트 케이스(권한 + 데이터 + UI 상태) 자동화 또는 체크리스트 통과
