## 3. 화면 / 상태 설계

| Screen | Route | 설명 |
| --- | --- | --- |
| `AnalysisLoadingScreen` | `AuthRoute.AnalysisLoading` | 결제 데이터 확인 -> 구독 분석 -> 리포트 정리 단계 진행 UI |

```kotlin
data class AnalysisLoadingUiState(
    val currentStep: AnalysisStep = AnalysisStep.FetchingPayment,
    val completedSteps: Set<AnalysisStep> = emptySet(),
    val savingAmount: Int = 0,
    val rotatingMessage: String = "",
    val isLoading: Boolean = true,
    val isPartialSuccess: Boolean = false,
    val error: AnalysisError? = null,
    val retryCount: Int = 0,
    val showExitDialog: Boolean = false,
)

enum class AnalysisStep {
    FetchingPayment,
    MatchingSubscriptions,
    BuildingReport,
}
```

### 필수 상태

- `Initial`: 1단계 활성, 금액 `0원`
- `Analyzing`: 현재 단계 강조, 절약 금액 카운트업
- `StepCompleted`: 완료 단계는 체크 아이콘 전환
- `Success`: 3단계 완료 후 `SCR-002b` 이동
- `PartialSuccess`: 미매핑 항목이 있어도 `SCR-002b`로 이동하고, 후속 Story3에서 확인 필요 배너로 이어짐
- `Error`: 재시도/종료 확인 제공

### 구현 포인트

- 단순 텍스트 3줄이 아니라 재사용 가능한 `ProgressRow`, `AmountCounterCard`, `AnalysisStatusBlock` 조합으로 설계한다.
- White Base, Primary 오렌지 제한 사용, 보더/타이포그래피 중심 계층을 따른다.
