# AEKKIM Android FE 초기세팅 가이드

> **기준 문서**: 요구사항정의서 v1.3 · 화면명세서 v1.3 · 바이브코딩_프로젝트_구조_설계서.md · BE 초기구조 분석서  
> **플랫폼**: Android (Kotlin + Jetpack Compose)  
> **아키텍처**: MVVM + 단방향 상태흐름 (MVI 경량)  
> **MVP 제한**: Room 제외 · 카카오 OAuth만 (FCM 딥링크/토큰 갱신 전략은 초기세팅에 포함)
> **반영 상태**: `FE_초기세팅_가이드_이슈_리포트.md` S1/S2 핵심 항목 반영 (2026-03-06)

> [중요] 이 문서는 reference-only 아카이브다. v1.5 현행 기준(구글 OAuth MVP, provider-neutral auth 구조)과 다를 수 있으며, 현재 구현 기준은 `FE/docs/2026-03-06_FE_초기세팅_작업_정리.md`를 따른다.

## 1. 프로젝트 생성 체크리스트

### 1-1. Android Studio 프로젝트 생성

- [ ] **Template**: Empty Activity
- [ ] **Package**: `com.ssafy.e106` (팀 컨벤션 통일)
- [ ] **Language**: Kotlin
- [ ] **Minimum SDK**: API 26 (Android 8.0) — UsageStats API 최소 요구사항
- [ ] **Build configuration language**: Kotlin DSL (`.kts`)

### 1-2. 프로젝트 생성 직후 필수 작업

- [ ] `.gitignore`에 `local.properties`, `*.jks` 추가 확인
- [ ] `local.properties`에 `BASE_URL_DEV`, `BASE_URL_PROD`, `KAKAO_NATIVE_APP_KEY` 추가
- [ ] Version Catalog (`libs.versions.toml`) 설정
- [ ] `build.gradle.kts` 의존성 추가
- [ ] `AndroidManifest.xml` 권한 선언
- [ ] `App.kt` (`@HiltAndroidApp`) + `MainActivity.kt` (`@AndroidEntryPoint`) 생성
- [ ] Kakao SDK 초기화
- [ ] Google Services `google-services.json` 추가 (FCM)

### 1-3. 이슈 리포트 반영 필수 항목 (착수 전)

- [ ] NotificationChannel 2종 생성 (`aekkim_checkin`, `aekkim_cancel_review`)
- [ ] `AuthInterceptor`에서 `runBlocking` 제거 (메모리 캐시 기반 토큰 조회)
- [ ] FCM 알림 탭 딥링크 라우팅 (`aekkim://checkin/...`, `aekkim://cancel/...`)
- [ ] Refresh Token 저장/재발급(`OkHttp Authenticator`) 전략 포함
- [ ] Release 난독화/리소스 축소(`isMinifyEnabled=true`, `isShrinkResources=true`) 적용
- [ ] 토큰 저장소 보안 강화(평문 저장 금지)

```kotlin
import android.content.Context

import androidx.security.crypto.MasterKey

        .build()

    private val prefs = EncryptedSharedPreferences.create(
        context,
        "auth_prefs_encrypted",
        masterKey,
        EncryptedSharedPreferences.PrefKeyEncryptionScheme.AES256_SIV,
        EncryptedSharedPreferences.PrefValueEncryptionScheme.AES256_GCM
    )

    fun getAccessToken(): String? = prefs.getString("access_token", null)
    fun getRefreshToken(): String? = prefs.getString("refresh_token", null)

    fun saveTokens(accessToken: String, refreshToken: String) {
        prefs.edit()
            .putString("access_token", accessToken)
            .putString("refresh_token", refreshToken)
            .apply()
    }

    fun clearTokens() {
        prefs.edit().clear().apply()
    }
}
```

```kotlin
package com.ssafy.e106.data.repository

import com.ssafy.e106.data.local.TokenDataStore
import java.util.concurrent.atomic.AtomicReference
import javax.inject.Inject
import javax.inject.Singleton

@Singleton
class TokenRepository @Inject constructor(
    private val tokenDataStore: TokenDataStore
) {
    private val accessTokenRef = AtomicReference<String?>(null)
    private val refreshTokenRef = AtomicReference<String?>(null)

    fun initialize() {
        accessTokenRef.set(tokenDataStore.getAccessToken())
        refreshTokenRef.set(tokenDataStore.getRefreshToken())
    }

    fun getAccessTokenSync(): String? = accessTokenRef.get()
    fun getRefreshTokenSync(): String? = refreshTokenRef.get()

    fun saveTokens(accessToken: String, refreshToken: String) {
        accessTokenRef.set(accessToken)
        refreshTokenRef.set(refreshToken)
        tokenDataStore.saveTokens(accessToken, refreshToken)
    }

    fun clearTokens() {
        accessTokenRef.set(null)
        refreshTokenRef.set(null)
        tokenDataStore.clearTokens()
    }
}
```

```kotlin
import com.ssafy.e106.data.repository.TokenRepository
import okhttp3.Interceptor
import okhttp3.Response
import javax.inject.Inject

class AuthInterceptor @Inject constructor(
    private val tokenRepository: TokenRepository
) : Interceptor {
    override fun intercept(chain: Interceptor.Chain): Response {
        val token = tokenRepository.getAccessTokenSync()
        val request = if (token != null) {
            chain.request().newBuilder()
                .addHeader("Authorization", "Bearer $token")
                .build()
        } else {
            chain.request()
        }
        return chain.proceed(request)
    }
}
```

### `data/auth/TokenAuthenticator.kt`

```kotlin
package com.ssafy.e106.data.auth

import com.ssafy.e106.data.api.AuthApi
import com.ssafy.e106.data.repository.TokenRepository
import okhttp3.Authenticator
import okhttp3.Request
import okhttp3.Response
import okhttp3.Route
import javax.inject.Inject
import javax.inject.Singleton

@Singleton
class TokenAuthenticator @Inject constructor(
    private val tokenRepository: TokenRepository,
    private val authApi: AuthApi
) : Authenticator {
    override fun authenticate(route: Route?, response: Response): Request? {
        if (response.code != 401) return null
        if (response.request.header("Authorization") == null) return null

        val refreshToken = tokenRepository.getRefreshTokenSync() ?: return null
        val newTokens = authApi.refreshTokenSync(refreshToken) ?: run {
            tokenRepository.clearTokens()
            return null
        }

        tokenRepository.saveTokens(newTokens.accessToken, newTokens.refreshToken)
        return response.request.newBuilder()
            .header("Authorization", "Bearer ${newTokens.accessToken}")
            .build()
    }
}
```

### 인증 에러 처리

- **401 Unauthorized** → `TokenAuthenticator`에서 refresh 재발급 시도
- refresh 실패 시 토큰 정리 후 `AuthNavGraph`로 이동
- **403 Forbidden** → 접근 권한 없음 → 사용자에게 안내 Snackbar

1. `UserApi.kt` — `GET /api/v1/me` 연동 (현재 가능, 인증 토큰 테스트용)

### 카카오 로그인 흐름

```kotlin
// AuthViewModel.kt 내부
fun onIntent(intent: AuthIntent) {
    when (intent) {
        is AuthIntent.KakaoLogin -> loginWithKakao(intent.context)
    }
}

private fun loginWithKakao(context: Context) = viewModelScope.launch {
    _state.update { it.copy(isLoading = true) }

    val oauthToken = suspendCancellableCoroutine { cont ->
        val callback: (OAuthToken?, Throwable?) -> Unit = { token, error ->
            if (error != null) cont.resumeWithException(error)
            else if (token != null) cont.resume(token)
        }
        if (UserApiClient.instance.isKakaoTalkLoginAvailable(context)) {
            UserApiClient.instance.loginWithKakaoTalk(context, callback = callback)
        } else {
            UserApiClient.instance.loginWithKakaoAccount(context, callback = callback)
        }
    }

    // 카카오 토큰 → BE JWT 교환
    authRepository.exchangeKakaoToken(oauthToken.accessToken)
        .onSuccess { jwt ->
            tokenRepository.saveTokens(
                accessToken = jwt.accessToken,
                refreshToken = jwt.refreshToken
            )
            _event.emit(AuthEvent.NavigateToHome)
        }
        .onFailure { e ->
            _state.update { it.copy(isLoading = false, error = e.message) }
        }
}
```

## 10. FCM 세팅

### `AekkimFirebaseMessagingService.kt`

```kotlin
package com.ssafy.e106.data.manager

import android.app.PendingIntent
import android.content.Intent
import androidx.core.app.NotificationManagerCompat
import androidx.core.app.NotificationCompat
import androidx.core.net.toUri
import com.google.firebase.messaging.FirebaseMessagingService
import com.google.firebase.messaging.RemoteMessage
import com.ssafy.e106.R
import com.ssafy.e106.app.MainActivity

class AekkimFirebaseMessagingService : FirebaseMessagingService() {

    override fun onNewToken(token: String) {
        super.onNewToken(token)
        // TODO: BE에 FCM 토큰 전송
    }

    override fun onMessageReceived(remoteMessage: RemoteMessage) {
        super.onMessageReceived(remoteMessage)
        val data = remoteMessage.data
        val type = data["type"]  // "CHECK_IN" or "PROMOTION"

        when (type) {
            "CHECK_IN" -> showCheckInNotification(
                subscriptionId = data["subscriptionId"] ?: return
            )
            "CANCEL_REVIEW" -> showCancelReviewNotification(
                subscriptionId = data["subscriptionId"] ?: return
            )
        }
    }

    private fun showCheckInNotification(subscriptionId: String) {
        val deepLinkIntent = Intent(
            Intent.ACTION_VIEW,
            "aekkim://checkin/$subscriptionId?isFullScreen=true".toUri(),
            this,
            MainActivity::class.java
        )
        val pendingIntent = PendingIntent.getActivity(
            this,
            subscriptionId.hashCode(),
            deepLinkIntent,
            PendingIntent.FLAG_UPDATE_CURRENT or PendingIntent.FLAG_IMMUTABLE
        )

        val notification = NotificationCompat.Builder(this, "aekkim_checkin")
            .setSmallIcon(R.drawable.ic_notification)
            .setContentTitle("체크인 알림")
            .setContentText("이번 달 이용 여부를 확인해주세요")
            .setContentIntent(pendingIntent)
            .setAutoCancel(true)
            .build()

        NotificationManagerCompat.from(this).notify(subscriptionId.hashCode(), notification)
    }

    private fun showCancelReviewNotification(subscriptionId: String) {
        val deepLinkIntent = Intent(
            Intent.ACTION_VIEW,
            "aekkim://cancel/$subscriptionId".toUri(),
            this,
            MainActivity::class.java
        )
        val pendingIntent = PendingIntent.getActivity(
            this,
            ("cancel_$subscriptionId").hashCode(),
            deepLinkIntent,
            PendingIntent.FLAG_UPDATE_CURRENT or PendingIntent.FLAG_IMMUTABLE
        )

        val notification = NotificationCompat.Builder(this, "aekkim_cancel_review")
            .setSmallIcon(R.drawable.ic_notification)
            .setContentTitle("해지 검토 알림")
            .setContentText("해지 검토가 필요한 구독이 있어요")
            .setContentIntent(pendingIntent)
            .setAutoCancel(true)
            .build()

        NotificationManagerCompat.from(this).notify(("cancel_$subscriptionId").hashCode(), notification)
    }
}
```

## 12. 바이브코딩 운영 규칙

### 필수 규칙 (어기면 PR 반려)

1. **Screen은 UI만** — Retrofit, Repository 직접 호출 금지
2. **상태 노출은 `StateFlow`만** — `MutableStateFlow`는 ViewModel 내부에서만
3. **액션은 `onIntent(XxxIntent)`로** — 직접 함수 호출 금지
4. **로딩/에러/빈 상태 모두 처리** — 3가지 상태 없으면 미완성
5. **새 기능 = 새 feature 폴더** — 기존 feature에 끼워넣기 금지

### AI 프롬프트 템플릿

```
[목표]
{기능명} 구현

[제약]
- Compose + MVVM 패턴 유지
- Screen은 UI만, 로직은 ViewModel/Repository로 분리
- 로딩/성공/실패 상태 모두 처리
- 바이브코딩 설계서 폴더구조 준수

[수정 범위]
- feature/{name}/view/{Name}Screen.kt
- feature/{name}/viewmodel/{Name}ViewModel.kt
- feature/{name}/viewmodel/{Name}State.kt
- feature/{name}/viewmodel/{Name}Intent.kt
- data/repository/{Name}Repository.kt (필요 시)

[완료 조건]
- 성공 시 UI 정상 노출
- 실패 시 ErrorScreen 또는 Snackbar
- 기존 동작 회귀 없음
```

### 주의사항 (Pitfall 방지)

| ❌ 하면 안 됨 | ✅ 올바른 방법 |
|-------------|--------------|
| `collectAsState()` 람다/중첩 내부에서 호출 | Composable 최상단에서 `collectAsStateWithLifecycle()` |
| `MutableStateFlow` 직접 노출 | `val state: StateFlow<...> = _state.asStateFlow()` |
| NavController를 ViewModel에 전달 | `() -> Unit` 람다를 Screen에서 받아 처리 |
| `as any` / 타입 캐스팅 억지 사용 | 올바른 타입 모델링 |
| `@ts-ignore` 류 억제 주석 | 근본적인 타입 문제 해결 |
| `runBlocking`을 ViewModel/Interceptor에서 사용 | ViewModel은 `viewModelScope`, Interceptor는 메모리 캐시 동기 조회 |

초기세팅 완료 판정은 "Gradle Sync 성공"만으로 끝내지 않고 아래 항목까지 확인한다.

```bash
./gradlew.bat :app:tasks
./gradlew.bat :app:assembleDebug
./gradlew.bat :app:compileDebugKotlin
```

- [ ] `:app:tasks` 성공
- [ ] `:app:assembleDebug` 성공
- [ ] `:app:compileDebugKotlin` 성공
