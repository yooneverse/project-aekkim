## 3. 현재 레포 기준 작업상황

- `AekkimFirebaseMessagingService`가 존재하지만 `onNewToken()`은 아직 실제 API 전송이 비어 있다.
- `HomeRoute.CheckIn`, `HomeRoute.CancelGuide` 및 딥링크 path가 선언돼 있다.
- `HomeNavGraph`의 체크인/해지 관련 화면은 placeholder 상태다.

즉, Story5는 골격은 일부 있으나 실제 API/화면/상태 흐름을 최신 기준으로 채워야 한다.

## 4. 화면 / 상태 설계

| Entry / Screen | Route | 설명 |
| --- | --- | --- |
| 체크인 배너 진입 | `HomeRoute.CheckIn(subscriptionId, isFullScreen=false)` | D-3 이후 미응답 |
| 체크인 풀스크린 진입 | `HomeRoute.CheckIn(subscriptionId, isFullScreen=true)` | 알림 탭 또는 D-3 당일 자동 노출 |
| 해지 지원 | `HomeRoute.CancelGuide(subscriptionId)` | `SCR-008` |
| 알림 토글 | `HomeRoute.MyPage` | 저장/원복 |

```kotlin
data class CheckInUiState(
    val subscriptionId: Long? = null,
    val serviceId: Long? = null,
    val serviceName: String = "",
    val cycleYm: String = "",
    val isFullScreen: Boolean = false,
    val isSubmitting: Boolean = false,
    val consecutiveBadCount: Int = 0,
    val error: String? = null,
)
```

### 필수 상태

- FCM 토큰 최초 등록 / 갱신 등록
- 알림 타입 분기
- cold start / warm start 딥링크 처리
- D-3 당일 풀스크린 1회 한정
- D-3 이후 미응답은 대시보드 배너만 노출
- 동일 월 중복 응답 시 최신 응답 저장 정책
- BAD 2회 연속 시 `SCR-007-2` 자동 분기

### 충돌 / 확인 필요 항목

- 알림 API 문서의 타입명은 `CHURN_REVIEW`인데 현재 FE 골격은 `CANCEL_REVIEW`를 사용하고 있어 `[확인 필요: FE/BE]`
- 화면명세에는 `해지 방법 보기`와 `해지하기`가 혼재하므로 버튼 라벨은 `[확인 필요: 기획]`
- pending checkin 조회 전용 API는 최신 API 문서에 명시돼 있지 않아 `[확인 필요: FE/BE]`

## 5. ViewModel / Repository / API 범위

### ViewModel 책임

`CheckInViewModel`

- 체크인 응답 버튼 선택 및 저장
- 후속 분기 여부 판단
- full-screen / banner 진입 차이 처리

`CancelGuideViewModel`

- 서비스별 텍스트 가이드 및 고객센터 전화하기 action 제공

`MyPageViewModel` 확장 범위

- 알림 토글 저장 / 실패 원복

### Repository 책임

- `NotificationRepository`: FCM 토큰 등록
- `CheckInRepository`: pending checkin 조회 `[확인 필요]`, 응답 저장, 최근 이력 조회
- `SettingsRepository`: 알림 설정 저장

### API 목록

| API | 상태 | 비고 |
| --- | --- | --- |
| `POST /api/v1/notifications/tokens` | 확정 | FCM 토큰 등록/갱신 |
| `PATCH /api/v1/users/me/notification-settings` | 확정 | 체크인/프로모션 알림 저장 |
| `POST /api/v1/checkins` | 확정 | `serviceId`, `cycleYm`, `response` 저장 |
| `GET /api/v1/checkins` | 확정 | 최근 이력/연속 BAD 판단 보조 |
| pending checkin 조회 API | `[확인 필요: FE/BE]` | 배너/자동 노출 판단 |
