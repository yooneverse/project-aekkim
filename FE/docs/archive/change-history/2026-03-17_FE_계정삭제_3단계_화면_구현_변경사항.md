# 계정 삭제 화면 구현 변경사항

> **작성일**: 2026-03-17  
> **최신 개정**: 2026-03-18  
> **범위**: `feature/settings/` 모듈 + `app/navigation/` 라우팅  
> **이전 동작**: AlertDialog 모달 1장 → 즉시 삭제  
> **변경 후 동작**: 전체 화면 2단계 플로우 (안내 → 최종 확인)

---

## 1. 개요

마이페이지 "계정 삭제" 진입 시 기존 `AlertDialog` 모달을 제거하고,  
별도 화면으로 이동하는 **2단계 계정 삭제 플로우**를 구현했다.

### 핵심 변경 요약

- 기존 MyPage 내 삭제 다이얼로그 → 독립 `DeleteAccount` 화면으로 분리
- 2단계 Step 전환 (Warning → Confirm) + AnimatedContent 슬라이드 애니메이션
- Warning: 간결한 삭제 의사 확인 / Confirm: 삭제 항목 상세 안내 + 동의 체크
- 기존 MVI 아키텍처 패턴 + Hilt DI 동일 적용
- API 호출은 기존 `UserRepository.deleteMe()` + `TokenRepository.clearTokens()` 재사용

## 3. 화면 흐름

### 3-1. 전체 네비게이션

```
MyPage → (계정 삭제 클릭) → DeleteAccount → (삭제 완료) → Onboarding
                              ↑ 이 화면       ← (뒤로가기) → MyPage
```

### 3-2. 2단계 내부 흐름

```
┌──────────────────────────────────────────────────────────┐
│                   DeleteAccountScreen                     │
│                                                          │
│  Step 1: Warning (안내)                                   │
│  ├─ 화면 중앙 경고 아이콘                                   │
│  ├─ "계정을 삭제하시려고요?"                                 │
│  ├─ "삭제하면 모든 데이터가 사라지고 복구할 수 없어요."         │
│  ├─ [삭제 진행] 버튼 → Step 2                              │
│  └─ [돌아가기] 버튼 → MyPage 복귀                          │
│                                                          │
│  Step 2: Confirm (최종 확인)                               │
│  ├─ "마지막으로 확인해 주세요"                               │
│  ├─ 삭제 항목 카드 (불렛 리스트 5종)                         │
│  ├─ 동의 체크박스 (체크 시 삭제 버튼 활성화)                  │
│  ├─ [계정 삭제] 버튼 → API 호출 → 온보딩 이동               │
│  └─ ← 뒤로 → Step 1                                      │
└──────────────────────────────────────────────────────────┘
```

### 3-3. 단계별 역할 분리

| 단계 | 목적 | 정보 밀도 |
|---|---|---|
| Warning | 삭제 의사 확인 (간결한 질문) | 최소 — 아이콘 + 한 줄 질문 + 부연 설명 |
| Confirm | 삭제 항목 상세 안내 + 최종 동의 | 상세 — 삭제 항목 5종 + 동의 체크 |

## 5. 핵심 구현 상세

### 5-1. ViewModel 단계 전환 로직

```
Intent         현재 Step     →  다음 Step        비고
──────────────────────────────────────────────────────────
NextStep       Warning       →  Confirm
NextStep       Confirm       →  (무시)
PreviousStep   Warning       →  NavigateBack     MyPage로 복귀
PreviousStep   Confirm       →  Warning          finalCheckAgreed 초기화
ConfirmDelete  Confirm       →  API 호출          finalCheckAgreed == true 필수
```

### 5-2. API 호출 흐름 (ConfirmDelete)

```
ConfirmDelete
  ├─ isDeleting || !finalCheckAgreed → return (가드)
  ├─ isDeleting = true
  ├─ userRepository.deleteMe()
  │   ├─ Success → tokenRepository.clearTokens()
  │   │            → NavigateToOnboarding("계정이 삭제되었어요")
  │   └─ Error   → ShowToast(에러 메시지)
  └─ isDeleting = false
```

### 5-3. 화면 전환 애니메이션

`AnimatedContent`로 Step 간 좌우 슬라이드 + 페이드 전환을 적용했다.

| 방향 | 진입 | 퇴장 |
|---|---|---|
| 다음 단계 (→) | slideInHorizontally(오른쪽→) + fadeIn | slideOutHorizontally(←왼쪽) + fadeOut |
| 이전 단계 (←) | slideInHorizontally(왼쪽→) + fadeIn | slideOutHorizontally(→오른쪽) + fadeOut |

### 5-4. 상단 프로그레스 바

`LinearProgressIndicator`로 현재 단계를 시각화한다.

| Step | Progress |
|---|---|
| Warning | 50% (1/2) |
| Confirm | 100% (2/2) |
