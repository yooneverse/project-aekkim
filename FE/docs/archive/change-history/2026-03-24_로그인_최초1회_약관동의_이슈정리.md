# 로그인 시 약관/구독분석 최초 1회 노출 이슈 정리

## 1) 배경
- 현상: 로그아웃 후 재로그인할 때마다 약관 동의 화면 및 구독 분석 진입 플로우가 반복 노출됨.
- 목표: **계정 생성 후 최초 로그인 1회에만** 약관/초기 분석 플로우를 수행하고, 이후 로그인에서는 대시보드로 바로 진입.

## 2) 기존 흐름 분석

### FE 진입/분기
- 로그인 성공 시 분기 위치: `FE/app/src/main/java/com/ssafy/e106/feature/auth/AuthViewModel.kt`
  - `loginResponse.isNewUser == true` → `NavigateToTerms`
  - `loginResponse.isNewUser == false` → `NavigateToDashboard`

### BE 판정/응답
- 소셜 로그인 사용자 upsert: `BE/src/main/java/com/ssafy/e106/domain/auth/service/OAuthService.java`
  - 기존 사용자 조회 성공 시 `isNewUser=false`
  - 신규 생성 시 `isNewUser=true`
- 로그인 응답 조립: `BE/src/main/java/com/ssafy/e106/domain/auth/service/AuthService.java`
- API 응답 노출: `BE/src/main/java/com/ssafy/e106/domain/auth/controller/AuthController.java`
  + `BE/src/main/java/com/ssafy/e106/domain/auth/dto/response/AuthLoginResponse.java`

## 3) 적용한 변경사항

### 3-1. BE: `isNewUser` 계약 보강
1. `OAuthService.upsertOAuthUser()` 반환형을 `UpsertOAuthUserResult(user, isNewUser)`로 변경
2. `AuthService.LoginResult`에 `isNewUser` 필드 추가
3. `AuthLoginResponse`에 `isNewUser` 필드 추가
4. `AuthController`에서 로그인 응답 생성 시 `isNewUser` 전달

#### 변경 파일
- `BE/src/main/java/com/ssafy/e106/domain/auth/service/OAuthService.java`
- `BE/src/main/java/com/ssafy/e106/domain/auth/service/AuthService.java`
- `BE/src/main/java/com/ssafy/e106/domain/auth/dto/response/AuthLoginResponse.java`
- `BE/src/main/java/com/ssafy/e106/domain/auth/controller/AuthController.java`

### 3-2. FE: 최초 로그인만 약관/분석 진입
1. `AuthUiEffect`에 `NavigateToDashboard` 추가
2. `AuthViewModel` 로그인 성공 분기 구현
   - 신규 사용자: Terms 이동
   - 기존 사용자: Dashboard 이동
3. `OnboardingRoute`, `AuthNavGraph`에 Dashboard 라우팅 연결

#### 변경 파일
- `FE/app/src/main/java/com/ssafy/e106/feature/auth/AuthContract.kt`
- `FE/app/src/main/java/com/ssafy/e106/feature/auth/AuthViewModel.kt`
- `FE/app/src/main/java/com/ssafy/e106/feature/auth/ui/OnboardingScreen.kt`
- `FE/app/src/main/java/com/ssafy/e106/app/navigation/AuthNavGraph.kt`

### 3-3. 재로그인 약관 재노출 방지 보조 가드(추가)
- 문제 재현 보고(재로그인 시 약관 재노출)를 반영해 FE에 방어 로직 추가
- `AuthOnboardingRepository`에 `hasRequiredConsentGranted()` 추가
- 최종 분기:
  - `shouldRunOnboarding = loginResponse.isNewUser && !hasRequiredConsentGranted()`
- 의미:
  - 서버에서 `isNewUser` 값이 흔들리더라도, 동일 디바이스에서 이미 약관 동의를 완료한 경우 재노출 방지

#### 변경 파일
- `FE/app/src/main/java/com/ssafy/e106/data/repository/AuthOnboardingRepository.kt`
- `FE/app/src/main/java/com/ssafy/e106/feature/auth/AuthViewModel.kt`

## 4) 검증 결과

### FE
- `./gradlew.bat testDebugUnitTest` → **성공**
- `./gradlew.bat assembleDebug` → **성공**

### BE
- `./gradlew.bat compileJava` → **성공**
- `./gradlew.bat test` → **실패 1건 (기존 환경 의존)**
  - `E106ApplicationTests > contextLoads()`
  - 원인: `FirebaseConfig` 초기화 관련 (`Firebase service account` 설정 의존)
  - 본 이슈(로그인 분기) 변경과 직접적 컴파일/로직 오류는 아님

## 5) 리스크 및 후속 권장사항
- 현재 보조 가드는 **디바이스 로컬 기준**으로 재노출을 막음.
- 정책이 "계정 단위로 약관 완료 상태를 서버에서 강제"라면,
  - 서버에 `onboardingCompleted`(또는 required terms agreed) 필드를 두고,
  - 로그인 응답에서 `needsOnboarding` 형태로 단일 권위 값 전달하는 방식이 가장 안전함.

## 6) 결론
- 요청사항인 "최초 로그인 1회만 약관/분석 수행"을 BE+FE 분기 구조로 반영함.
- 추가로 로컬 보조 가드를 넣어, 실제 현상(재로그인 시 반복 노출)까지 즉시 완화함.
