# 2026-03-23 FE MVP UI 수정사항 정리

## 1. 목적

프론트 1차 MVP 테스트에서 확인된 UI/네비게이션 이슈 중, 2026-03-23 기준으로 수정한 항목을 정리한다.

## 2. 수정 항목

### 2.1 체크인 알림 클릭 시 이동 경로 수정

- 대상: 알림함
- 기존 문제:
  - `CHECKIN` 알림을 누르면 체크인 화면이 아니라 구독 상세 화면으로 이동함
- 수정 내용:
  - 알림함 클릭 시 `CHECKIN` 알림은 체크인 화면으로 이동하도록 분기 수정
  - 푸시 클릭 동작과 알림함 클릭 동작을 동일하게 맞춤

### 2.2 구독 상세 진입 및 구독 수정 흐름 수정

- 대상: 대시보드, 구독 상세 화면
- 기존 문제:
  - 대시보드에서 구독 카드 클릭 시 상세 진입 흐름이 일관되지 않음
  - 구독 상세 화면에서 `구독수정하기` 버튼 클릭 시 수정 화면이 아니라 다른 화면 흐름으로 빠짐
- 수정 내용:
  - 대시보드 구독 카드 클릭 시 구독 상세 route로 진입하도록 통일
  - 구독 상세 화면에서 `구독수정하기` 클릭 시 대시보드의 기존 수정 바텀시트를 열도록 연결

### 2.3 바텀시트 부분 확장 문제 수정

- 대상: 공통 바텀시트
- 기존 문제:
  - 바텀시트가 중간 높이에서 멈춘 것처럼 보여 사용성이 떨어짐
- 수정 내용:
  - 공통 바텀시트에서 부분 확장 상태를 사용하지 않도록 변경

### 2.4 구독 결제일 선택 UI 개선

- 대상: 구독 추가/수정 바텀시트
- 기존 문제:
  - `1 ~ 31` 버튼 전체 노출 방식이 투박하고 현재 선택값 인지가 어려움
- 수정 내용:
  - 결제일 선택 방식을 `NumberPicker` 기반 휠 피커로 변경
  - 상단에 현재 선택된 결제일 요약을 함께 표시

### 2.5 카드 프로모션 summary 가독성 개선

- 대상: 추천 리스트, 프로모션 상세
- 기존 문제:
  - 카드 혜택 summary가 줄 단위 정보임에도 일반 문장처럼 보여 가독성이 낮음
- 수정 내용:
  - `CARD_BENEFIT` 타입에 한해 summary를 줄 단위로 렌더링
  - `연회비`, `국내전용`, `해외겸용`, `전월실적`, `브랜드` 라벨을 강조 표시

### 2.6 앱 런처 아이콘 표시 수정

- 대상: 앱 아이콘
- 기존 문제:
  - 런처 아이콘이 표시되지 않음
- 수정 내용:
  - `AndroidManifest.xml`에 앱 아이콘/라운드 아이콘 리소스를 연결

### 2.7 서비스 로고 상대경로 처리 수정

- 대상: 대시보드, 구독 상세, 체크인, 취소 가이드, 프로모션 내 서비스 로고
- 기존 문제:
  - 백엔드가 `/assets/services/...` 형식의 상대경로를 내려줄 때 로고가 표시되지 않을 수 있음
- 수정 내용:
  - FE repository 레이어에서 상대경로를 `BASE_URL` 기준 절대경로로 변환하도록 보정

### 2.8 해지 가이드 링크 표시 방식 개선

- 대상: 해지 방법 보기 화면
- 기존 문제:
  - 긴 URL 전체가 그대로 노출되어 화면상 보기 좋지 않음
- 수정 내용:
  - 실제 링크 값은 유지하고, 화면에는 `host + path` 형태의 축약 텍스트를 표시

### 2.9 구독 추가 플로우 바텀시트 단일화

- 대상: 구독 추가 플로우
- 기존 문제:
  - 서비스 선택 바텀시트와 요금제/결제일 바텀시트가 따로 떠서 흐름이 끊겨 보임
- 수정 내용:
  - 서비스 선택과 요금제/결제일 설정을 하나의 바텀시트 안에서 단계만 전환되도록 변경
  - 사용자가 같은 시트 안에서 자연스럽게 다음 단계로 이동하도록 개선

## 3. 재확인 필요 항목

- 앱 아이콘 표시 여부
- 서비스 로고 표시 여부
- 알림함에서 `CHECKIN` 클릭 시 체크인 화면 이동 여부
- 대시보드 -> 구독 상세 -> 구독 수정 흐름
- 구독 추가 시 단일 바텀시트 안에서 서비스 선택 -> 요금제/결제일 설정 -> 완료 흐름
- 카드 프로모션 summary 라벨 강조 표시 여부
- 해지 가이드 링크 축약 표시 여부

## 4. 비고

- 일부 AndroidManifest XML 경고는 IDE schema 인식 경고이며, 실제 빌드 오류는 아님
- 위 수정사항 반영 후 `:app:compileDebugKotlin` 컴파일은 통과함
