# S14P21E106-230 대시보드 네비게이션 및 구독 수정 플로우 버그 수정

## 배경

- 대시보드/추천 탭 전환 중 대시보드 복귀가 불안정한 문제가 있었다.
- 대시보드 구독 카드 탭 시 구독 상세 페이지로 이동하지 않는 문제가 있었다.
- 구독 상세에서 `구독 수정하기` 클릭 시 수정 UI가 열리지 않는 문제가 있었다.

## 원인 요약

1. 대시보드 복귀를 `navigate(HomeRoute.Dashboard)` 중심으로만 처리해 백스택 상황에 따라 기대한 복귀 동작이 보장되지 않았다.
2. 대시보드 카드 클릭이 상세 라우트 이동으로 연결되지 않고 로컬 상태 플로우로만 연결되어 있었다.
3. 상세 화면의 `구독 수정하기` 액션이 편집 인텐트로 이어지지 않고 dismiss만 수행하고 있었다.

## 수정 내용

### 1) 대시보드 복귀 안정화

- 파일: `FE/app/src/main/java/com/ssafy/e106/app/navigation/HomeNavGraph.kt`
- `navigateToDashboardRoot()`에서 아래 순서로 처리하도록 변경
  - `popBackStack(HomeRoute.Dashboard, inclusive = false)` 우선 시도
  - 실패 시 기존 `navigate(HomeRoute.Dashboard)` fallback 수행

### 2) 대시보드 → 구독 상세 라우팅 복구

- 파일: `FE/app/src/main/java/com/ssafy/e106/feature/dashboard/ui/DashboardRoute.kt`
  - `onNavigateToSubscriptionDetail: (Long) -> Unit` 콜백 추가
  - 카드 클릭 이벤트를 네비게이션 콜백으로 직접 연결
- 파일: `FE/app/src/main/java/com/ssafy/e106/app/navigation/HomeNavGraph.kt`
  - `DashboardRoute`에 `HomeRoute.SubscriptionDetail` 라우팅 콜백 전달

### 3) 구독 상세 → 수정 바텀시트 플로우 복구

- 파일: `FE/app/src/main/java/com/ssafy/e106/feature/dashboard/ui/SubscriptionDetailRoute.kt`
  - `onStartEditSubscription: (Long) -> Unit` 콜백 추가
  - `onEditSubscription`을 dismiss가 아닌 편집 시작 콜백으로 연결
- 파일: `FE/app/src/main/java/com/ssafy/e106/app/navigation/HomeNavGraph.kt`
  - 상세 화면에서 편집 클릭 시 `previousBackStackEntry.savedStateHandle`에 수정 대상 ID 저장 후 `popBackStack()`
  - 대시보드 엔트리에서 pending edit id를 수신해 `DashboardRoute`로 전달
- 파일: `FE/app/src/main/java/com/ssafy/e106/feature/dashboard/ui/DashboardRoute.kt`
  - pending edit id 감지 시 `DashboardIntent.StartEditSubscription(id)` 실행 후 키 소비
