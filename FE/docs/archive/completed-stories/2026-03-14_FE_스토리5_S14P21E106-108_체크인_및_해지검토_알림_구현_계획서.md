# FE 스토리5 - S14P21E106-108 체크인 및 해지검토 알림 구현 계획서

> 최초 작성: 2026-03-09  
> 최신 개정: 2026-03-15  
> 상태 요약: `진행 중`

## 1. 문서 목적

- 체크인, 해지 검토, 해지 가이드, FCM 토큰 등록, 알림함 목록/읽음 처리/라우팅을 하나의 스토리로 재정의한다.
- 최근 30일 UsageStats를 하루 1회 판정해 `lowUsageDetected`만 서버에 전달하고, `lowUsageDetected == true` 인 구독만 CHECKIN/PROMO 알림 후보로 사용한다.

## 2. 최신 범위

- 포함
  - 체크인 화면 `SCR-007`
  - 해지 검토 후속 `SCR-007-2`
  - 해지 가이드 `SCR-008`
  - FCM 토큰 등록
  - 알림함 목록/읽음 처리/타입별 라우팅
- 제외
  - 고도화된 알림 생성 배치 규칙

## 3. 선행조건

- 알림 타입 정본은 `CHECKIN`, `PROMO`, `CHURN_REVIEW`다.
- 알림 payload 식별자는 `referenceId`다.
- 현재 BE는 알림 도메인 API가 미구현 상태다.
- FE는 UsageStats 원본이 아니라 하루 1회 계산한 최근 30일 사용량 판정값만 서버로 전달한다.

## 4. 화면 범위

- `SCR-007` 체크인
- `SCR-007-2` 해지 검토 후속
- `SCR-008` 해지 가이드
- `[신규 화면 정의 필요]` 알림함 목록

## 6. 데이터 모델

- Check-in item: `subscriptionId`, 서비스명, 질문 상태, 응답 결과
- Churn review: 최근 체크인 이력, 절약 금액, 다음 화면 분기
- Cancel guide: 고객센터 전화, 이메일, 해지 URL
- Notification item: `notificationId`, `type`, `referenceId`, `title`, `body`, `isRead`, `readAt`, `sentAt`

## 8. 태스크 분해

| 태스크 | 현재 상태 | FE 단독 가능 여부 | 근거 | 권장 조치 |
| --- | --- | --- | --- | --- |
| FCM 서비스 정리 | 진행 중 | 가능 | 서비스와 채널은 있으나 `onNewToken()`이 TODO이고 enum 규약이 구버전이다. | enum/payload 규약을 최신 기준으로 교체한다. |
| FCM 토큰 등록 | 미완 | 불가 | API가 아직 없다. | contract-first 문서와 호출 시점을 먼저 확정한다. |
| 체크인 화면 | 미완 | 가능 | route가 플레이스홀더다. | mock 응답 기준으로 실제 화면을 먼저 만든다. |
| 해지 검토 후속 화면 | 미완 | 가능 | 현재 구현이 없다. | `거의 안 썼어요` 연속 조건 분기 화면을 신규 추가한다. |
| 해지 가이드 | 미완 | 부분 가능 | route는 있으나 placeholder다. | `tel:`/URL CTA가 가능한 실제 화면으로 교체한다. |
| 알림함 목록/읽음 처리 | 미완 | 가능 | SRS에는 있으나 화면/코드 모두 부재다. | 신규 route와 mock 리스트 화면을 먼저 추가한다. |

## 9. 완료 기준

- 알림 enum/payload 규약이 최신 SRS와 일치한다.
- 체크인, 해지 검토, 해지 가이드가 플레이스홀더가 아니다.
- 알림함 목록과 읽음 처리 흐름이 문서/코드에서 정의된다.
- FCM 토큰 등록 시점과 실패 재시도 정책이 문서에 고정된다.
- 최근 30일 UsageStats 기반 `lowUsageDetected`를 하루 1회 계산해 서버에 기록하는 흐름이 문서에 고정된다.
- CHECKIN/PROMO 알림 후보는 `lowUsageDetected == true` 기준을 전제로 한다.
