  - 프로필 조회
  - 체크인 알림 토글
  - 프로모션 알림 토글
  - 선택 정보 활용 동의 토글
  - 약관/개인정보/문의 링크
  - 로그아웃
  - 회원탈퇴
- 제외
  - 소셜 연동 해제 독립 기능

## 3. 선행조건

- 사용자 API와 인증 API가 최신 계약으로 동작해야 한다.
- provider 배지는 화면명세상 필수지만 현재 API에는 필드가 없다. 이 항목은 `[BE 확정 필요]`다.

## 4. 화면 범위

- `SCR-010` 마이페이지

## 5. 연동 API

- `GET /api/v1/users/me`
- `PATCH /api/v1/users/me/notification-settings`
- `PATCH /api/v1/users/me/optional-consent`
- `POST /api/v1/auth/logout`
- `DELETE /api/v1/users/me`

## 6. 데이터 모델

- 프로필: 이미지, 닉네임, 이메일, provider 배지
- 설정: `checkinAlertEnabled`, `promoAlertEnabled`, `optionalConsentAgreed`
- 계정 관리: 로그아웃/탈퇴 처리 상태, 확인 다이얼로그 상태

## 8. 태스크 분해

| 태스크 | 현재 상태 | FE 단독 가능 여부 | 근거 | 권장 조치 |
| --- | --- | --- | --- | --- |
| 내 정보 조회 | 진행 중 | 부분 가능 | `GET /users/me` 연동은 되어 있으나 provider와 optional consent 노출이 부족하다. | UI 모델을 확장하고 누락 필드를 반영한다. |
| 체크인/프로모션 알림 토글 저장 | 미완 | 부분 가능 | 현재 토글은 로컬 상태만 바뀐다. | PATCH 연동, 로딩, 실패 롤백을 구현한다. |
| 선택 동의 토글 저장 | 미완 | 부분 가능 | 화면/현재 코드에 존재하지 않는다. | 신규 토글과 PATCH를 추가한다. |
| 로그아웃 | 완료 | 부분 가능 | API 호출, 토큰 삭제, 온보딩 복귀가 구현되어 있다. | 완료 유지, 다이얼로그 문구와 provider 하드코딩만 보완한다. |
| 회원탈퇴 | 완료 | 부분 가능 | DELETE와 세션 정리가 구현되어 있다. | 완료 유지, 에러 케이스 QA만 보강한다. |
| 외부 링크 | 미완 | 가능 | URL이 비어 있어 준비중 토스트만 보인다. | 채널 확정 전까지 placeholder 정책을 문서화한다. |

## 9. 완료 기준

- 프로필 조회가 실제 API 응답과 동기화된다.
- 세 개의 토글이 모두 즉시 저장되고 실패 시 복구된다.
- 로그아웃/회원탈퇴 다이얼로그와 후속 세션 정리가 정상 동작한다.
- provider 정보가 확정되기 전까지 하드코딩을 완료로 보지 않는다.

## 10. QA 포인트

- 최초 진입 로딩과 재시도
- 토글 저장 중 중복 탭 방지
- 저장 실패 시 이전 상태 복구
- 로그아웃 후 재진입 시 세션 초기화 확인
- 탈퇴 후 재로그인 플로우 확인
