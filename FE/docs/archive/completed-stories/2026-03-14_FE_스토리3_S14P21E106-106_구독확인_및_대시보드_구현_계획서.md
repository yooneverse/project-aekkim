# FE 스토리3 - S14P21E106-106 구독확인 및 대시보드 구현 계획서

> 최초 작성: 2026-03-09  
> 최신 개정: 2026-03-15  
> 상태 요약: `진행 중`

## 1. 문서 목적

- 구독 확인, 대시보드, 수동 구독 추가, 구독 상세, 확인 필요 결제 흐름을 최신 화면명세와 현재 BE 상태 기준으로 재정의한다.
- 분석 완료 직후 `SCR-002b`에서는 감지된 구독 목록과 확인 필요 결제 요약을 함께 보여주고, 상세 판단은 `SCR-009`로 넘긴다.

## 2. 최신 범위

- 포함
  - `SCR-002b` 구독 확인
  - `SCR-003` 대시보드
  - `SCR-003-2` 서비스 선택
  - `SCR-003-3` 요금제/결제일 설정
  - `SCR-004` 구독 상세
  - `SCR-009` 확인 필요 결제
- 제외
  - 프로모션 목록/상세는 Story3 연계 대상이지만 구현 자체는 별도 태스크로 관리

## 3. 선행조건

- Story2에서 UsageStats 권한이 허용되어야 한다.
- BE 구독 API는 현재 미구현 상태다.
- 따라서 본 스토리는 `mock UI 완성 + contract-first 연동 준비`를 기본 전략으로 한다.

## 9. 완료 기준

- 구독 확인과 대시보드 진입이 플레이스홀더가 아니다.
- `SCR-002b`에서 감지 구독 목록과 확인 필요 결제 요약이 함께 노출된다.
- 서비스 선택은 OTT 6종을 기준으로 노출된다.
- 구독 상세에서 전화/해지 가이드/추천 이동을 위한 데이터 슬롯이 정의된다.
- 확인 필요 결제 화면의 제외/추가/확정 플로우가 화면명세와 일치한다.
- `구독 아님`은 현재 세션 로컬 상태에서만 제외되고, 서버에는 확정 항목만 전달된다.
- mock 데이터와 real DTO 전환 지점이 문서에 고정된다.

## 10. QA 포인트
