# E106 외부 서비스 연동 정보

프로젝트에서 사용하는 외부 서비스와 설정 위치를 정리한 문서입니다.

## 1. 소셜 인증

### 1.1 Google OAuth

**사용처**
- FE: Google 로그인, ID Token 획득
- BE: ID Token `aud` 검증

**준비 항목**
- Google Cloud Console 프로젝트 생성
- Android OAuth Client 생성
- Web Client 생성

**설정 위치**
- FE: `FE/local.properties` -> `GOOGLE_WEB_CLIENT_ID`
- BE: `.env*` -> `GOOGLE_CLIENT_ID`

**체크 포인트**
- FE Web Client ID와 BE 검증용 Client ID가 일치해야 함
- SHA-1, SHA-256 인증서 지문 등록 여부 확인

### 1.2 Kakao Login

**사용처**
- FE: 카카오 로그인

**준비 항목**
- Kakao Developers 앱 생성
- Android 플랫폼 등록

**설정 위치**
- FE: `FE/local.properties` -> `KAKAO_NATIVE_APP_KEY`

**체크 포인트**
- `AndroidManifest` 내 카카오 스킴 `kakao{APP_KEY}` 설정 확인

## 2. Firebase / FCM

**사용처**
- FE: 푸시 수신, 토큰 발급
- BE: Firebase Admin 기반 푸시 발송

**준비 항목**
- Firebase 프로젝트 생성
- Android 앱 등록
- 서비스 계정 JSON 발급

**설정 위치**
- FE: `FE/app/google-services.json`
- BE: `.env*` -> `FIREBASE_SERVICE_ACCOUNT_JSON_BASE64`

**체크 포인트**
- 서비스 계정 JSON 원문을 Base64로 인코딩해 BE 환경변수에 주입
- 앱 패키지명과 인증서 SHA 값이 Firebase 설정과 일치해야 함

## 5. 보안 / 운영 주의사항

- 저장소에 토큰, 앱 키, 클라이언트 ID 등 민감 정보가 커밋되지 않도록 관리
- 이미 노출된 키는 즉시 폐기 후 재발급
- `dev`, `prod` 환경의 키와 프로젝트는 분리 운영
- CI 로그에 민감 정보가 출력되지 않도록 `set +x`, 마스킹 설정 등을 확인
