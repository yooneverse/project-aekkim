# E106 시연 시나리오

## 0. 시연 환경
- API 기준 주소: `https://api.example.com`
- 시연 디바이스: Android 실기기 1대
- 계정: Google 또는 Kakao 테스트 계정
- 데이터 상태:
  - 서비스/번들/프로모션 seed 반영
  - 테스트 사용자 결제내역 및 구독 데이터 존재

## 1. 앱 시작 및 온보딩
1. 앱 아이콘 탭
2. 온보딩 화면에서 `시작하기` 버튼 탭
3. 권한 안내(UsageStats/알림) 확인 후 `동의하고 진행` 탭

## 2. 소셜 로그인
1. 로그인 화면에서 `Google로 계속하기` 또는 `카카오로 계속하기` 버튼 탭
2. 계정 선택/인증 완료
3. 로그인 성공 후 구독 확인(분석) 진입

## 3. 구독 확인(분석) 플로우
1. `구독 확인 시작` 버튼 탭
2. 로딩 화면(온디바이스 분석 + 서버 연동) 확인
3. 분석 결과에서 탐지된 구독 후보 목록 확인
4. 항목별 토글/선택 후 `확정` 버튼 탭

## 4. 대시보드
1. 하단 탭 `홈` 진입
2. 상단 요약 카드 확인
  - 월 구독 총액
  - 절감 가능 금액
3. 구독 카드 중 하나 탭 -> 상세 진입

## 5. 구독 상세 및 체크인
1. 구독 상세 화면에서 `이번 달 사용 확인` 버튼 탭
2. 체크인 응답 선택
  - `잘 사용 중`
  - `거의 안 써요`
  - `모르겠어요`
3. 저장 후 체크인 이력 반영 확인

## 6. 프로모션 추천
1. 하단 탭 `추천` 진입
2. 추천 목록에서 카드/프로모션/번들 카드 확인
3. 카드 1개 탭 -> 상세 화면 진입
4. `바로가기` 링크 탭(외부 링크 이동)

## 7. 알림
1. 하단 탭 또는 우상단 벨 아이콘 탭
2. 알림 목록에서 미읽음 항목 탭
3. 읽음 처리 및 리스트 반영 확인

## 8. 마이페이지
1. 하단 탭 `마이` 진입
2. 알림 수신 설정 On/Off 변경
3. 선택 동의(마케팅) 화면 진입/수정
4. 약관/개인정보처리방침 화면 열람

## 9. 장애 대응 시연(백오피스/운영자)
1. `/admin` 접근 후 seed import 또는 배치 실행
2. 특정 배치(체크인/프로모션) 실행
3. 알림 생성 여부 확인

## 10. 시연 체크리스트
- 로그인 성공 및 JWT 갱신 정상
- 구독 분석 결과가 빈 화면이 아닌지 확인
- 체크인 저장 후 대시보드 재조회 반영
- 프로모션 상세 링크 이동 정상
- 알림 읽음 처리 상태 변경 정상
- 서버 헬스체크 `/api/v1/health` 200 확인
