## 2. 클론 이후 빌드/실행 절차

### 2.1 공통
1. 저장소 클론
2. 루트에서 환경파일 생성

```bash
cp .env.example .env
```

3. 필수 민감값 입력
- `JWT_SECRET`
- `GOOGLE_CLIENT_ID`
- `FIREBASE_SERVICE_ACCOUNT_JSON_BASE64`
- `FINANCE_API_KEY`
- (필요 시) DB 비밀번호

### 2.2 로컬 백엔드 스택(Docker)

```bash
docker compose up -d --build
```

- 기동 서비스: `mysql`, `redis`, `backend`
- 백엔드 기본 포트: `${BACKEND_PORT}` (기본 8080)

### 2.3 백엔드 단독 실행(선택)

```bash
cd BE
./gradlew bootRun
```

### 2.4 안드로이드 앱 빌드
1. `FE/local.properties` 생성(또는 `local_ondevice.properties` 참고)
2. `FE/app/google-services.json` 배치
3. 실행

```bash
cd FE
./gradlew installDebug
```

추가 빌드:
```bash
./gradlew assembleDevice
./gradlew assembleRelease
./gradlew bundleRelease
```

## 4. 배포 시 특이사항

### 4.1 브랜치별 배포 정책(Jenkins)
- `develop` -> `dev` 배포 (`INFRA/deploy-dev.sh`)
- `main` -> `prod` 배포 (`INFRA/deploy-prod.sh`)
- 그 외 브랜치는 배포 차단

### 4.2 자동 롤백
- prod 배포 실패 시 `prod-prev` 이미지로 자동 롤백 시도
- 수동 롤백: `INFRA/rollback.sh`

### 4.3 포트/라우팅
- dev backend: 8081
- prod backend: 8082
- Nginx:
  - `/api/v1/*`, `/admin*` -> prod(8082)
  - `/dev/api/v1/*`, `/dev/admin*` -> dev(8081)

### 4.4 DB 스키마 모드
- local/dev: `DDL_AUTO=update`
- prod: `DDL_AUTO=validate`
